# 바우처 알리미 — 에너지바우처 자격 자동탐지·원클릭 신청

> **아이디어 간략 개요 (3줄 이내)**
> 에너지바우처 지원 대상임에도 신청주의(申請主義) 구조 때문에 혜택을 받지 못하는 약 17만 취약가구를 전력·에너지 공공데이터로 자동 발굴하고, AI 챗봇이 원클릭 신청까지 안내하는 복지 사각지대 해소 서비스다.
> 한국전력 계약종별 전력사용량, 에너지공단 에너지통계·신재생보급 데이터를 결합해 저사용량·고요금 패턴 가구를 식별하고, 행정정보 연계로 자격 추정·증빙 자동 수집·신청서 대행 작성까지 원스톱 제공한다.

**핵심 기술·서비스·정보 명칭**

| 구분 | 명칭 |
|:---|:---|
| 핵심 서비스 | 바우처 알리미 (Voucher Finder) |
| 자격 추정 엔진 | 취약가구 자격 추정 AI 모델 (Random Forest + 규칙 앙상블) |
| 신청 지원 | 에너지복지 원클릭 신청 챗봇 (LLM 기반 대화형 안내) |
| 핵심 데이터 | 한국전력 계약종별 전력사용량 / 에너지공단 100대 에너지통계 / 기초지자체별 신재생 보급현황 |

## 1. 아이디어 기획 핵심내용 (구체성, 우수성)

### 1.1 무엇을 만드는가

**바우처 알리미**는 에너지바우처 복지 사각지대를 해소하는 세 가지 기능을 결합한 공공데이터 기반 서비스다.

1. **자격 자동탐지**: 한국전력 계약종별 전력사용량(API, `15101360`) 데이터를 기초 지자체별로 집계해, 저소득 요금제(주택용 저압 소규모) 구간에서 비정상적 고요금 패턴이 발생하는 가구 밀도를 지역 단위로 식별한다. 에너지공단 100대 에너지통계(`15121342`)의 가구별 에너지소비 분포, 기초지자체별 신재생 보급현황(`15086292`)을 결합해 지역 에너지 취약도 지수를 산출한다.

2. **개인 자격 확인**: 사용자가 주소와 가구 정보를 입력하면 AI 자격 추정 모델이 에너지바우처·에너지효율화 지원 등 복지 프로그램 해당 여부를 즉시 알려준다. 에너지바우처 법정 수급 자격(기초생활·차상위·장애인·노인·영유아 가구 등)을 규칙 기반으로 1차 필터링한 뒤, 데이터 패턴 기반 추정 모델로 2차 보완한다.

3. **원클릭 신청 챗봇**: LLM 기반 챗봇이 자격 확인부터 필요 서류 안내, 읍·면·동 주민센터 온라인 신청 페이지 연결, 신청서 초안 자동 작성까지 단계별로 안내한다. 노인·장애인 가구를 위해 고령친화 UI(큰 글씨, 음성 안내)를 제공한다.

### 1.3 우수성 — 기존 대비 핵심 개선

기존 에너지바우처 제도는 수급권자가 직접 신청해야 하는 **신청주의** 구조다. 에너지복지포털(한국에너지공단 운영)은 단순 정보 제공·신청 접수 창구에 그치며, 지원 대상인지 여부를 능동적으로 알려주는 기능이 없다. 특히 노인·장애인 가구는 디지털 정보 접근성이 낮아 제도 존재 자체를 모르는 경우가 많다.

바우처 알리미는 **공급자 중심(신청 처리) → 수요자 중심(대상자 발굴·안내)**으로 패러다임을 전환한다. 지자체 담당 공무원이 취약지역 현황 지도를 활용해 방문상담 우선순위를 정하는 B2G 활용, 복지관·경로당을 통한 B2C 접점 확대까지 포함한다.

## 2. 아이디어 구상 및 제안배경 (활용적정성)

### 2.1 현황 및 문제점

**에너지바우처 사각지대 문제**

에너지바우처는 저소득 취약계층 가구에 냉·난방비를 지원하는 산업통상자원부 소관 복지 사업이다. 2023년 기준 지원 대상 130.7만 가구 중 실지급 가구는 113.6만 가구로,[^1][^2] **약 17만 가구(13%)가 대상임에도 혜택을 받지 못하는 사각지대**에 놓여 있다. 미수급의 주요 원인은 신청주의 구조 — 자격이 있어도 신청하지 않으면 지원이 이루어지지 않는다.

에너지 취약계층 문제는 경제적 피해에 그치지 않는다. 2022년 폭염 사망자의 다수는 에어컨이 없거나 전기 요금 부담으로 가동을 자제한 고령·저소득 가구였으며,[^3] 한파 시기 난방 중단으로 인한 한랭 질환도 유사한 패턴을 보인다. 에너지 빈곤(Energy Poverty)은 단순 경제 문제가 아닌 보건·생명 문제다.

### 2.2 활용 4요소

| 요소 | 내용 |
|:---|:---|
| **활용분야** | 에너지 복지 행정 + 사회안전망 강화. 산업부 소관 에너지바우처 제도의 미수급 문제 해소, 지자체 복지 담당 공무원 업무 보조, 복지관·경로당 현장 활용 |
| **활용빈도** | 연 2회 정기(하계 6~8월·동계 10~3월) 신청 기간에 집중 + 상시 자격 확인. 기초지자체 약 228개 단위로 월별 데이터 갱신 시 취약도 지수 재산출 |
| **활용범위** | 전국 기초지자체 228개 행정 단위, 에너지바우처 대상 약 130만 가구 직접 대상. 지자체 복지부서·읍면동 주민센터(약 3,500개)·복지관·경로당·장애인 시설 등 중간 전달 채널 포함 |
| **중요성** | 에너지 취약계층 17만 가구 사각지대 해소 → 1가구당 평균 지원액 약 20만원[^4] 기준 연간 340억원 규모의 복지 누수 방지. 폭염·한파 사망 예방 등 사회적 비용 절감 효과. 산업부 에너지 복지 정책의 실효성 제고 |

### 2.3 경영혁신·창업학적 프레임워크

**JTBD(Jobs To Be Done) 및 서비스 설계**

에너지바우처 사각지대 문제를 JTBD 관점으로 분석하면, 취약계층 가구가 완수하려는 과제(Job)는 "에너지 요금 부담을 줄이고 안전하게 냉난방을 유지하는 것"이다. 그러나 신청주의 구조, 행정 정보 부족, 디지털 접근성 한계가 **기능적·사회적·감정적 장벽**으로 작용해 과제 완수를 막는다.

바우처 알리미는 이 장벽을 제거하는 **과제-솔루션 정합(Job-Solution Fit)** 을 목표로 한다. 단순 정보 제공이 아니라 "발굴 → 자격 확인 → 신청 완료"까지 과제 전체를 완수할 수 있도록 엔드-투-엔드 지원을 설계한다.

**파괴적 혁신 관점 (Christensen)**

에너지바우처 제도는 복지 공무원·복지관 상담원이 개별 안내를 담당하는 고비용 구조다. 바우처 알리미는 공공데이터 + AI를 결합해 저비용으로 동일한(또는 더 넓은) 도달 범위를 달성하는 **저가형 파괴적 혁신** 모델이다. 기존 서비스(복지 상담)을 대체하는 것이 아니라 기존에 닿지 못했던 사각지대를 커버한다.

### 3-④ 창의성·독창성

**① 신청주의 → 발굴주의 패러다임 전환**

국내외 복지 제도의 대부분은 수급권자가 자신의 자격을 알고 신청해야 하는 신청주의 구조다. 행정 비용 효율화를 위한 구조적 선택이지만, 정보 취약 계층은 제도 존재 자체를 모르거나 신청 절차의 복잡성으로 인해 배제된다. 바우처 알리미는 **공공데이터를 활용해 대상자를 시스템이 먼저 찾아가는** 역방향 복지 설계를 구현한다.

**② 에너지 공공데이터를 복지 발굴에 재목적화(Repurpose)**

한국전력 계약종별 전력사용량 데이터는 원래 전력 수요 관리·요금 정산용으로 수집된다. 이 데이터를 **취약계층 에너지 사용 패턴 식별**에 재활용하는 것이 핵심 창의성이다. 주택용 저압 계약의 저소득 밀도가 높은 지역에서 계절별 전력 소비가 비정상적으로 낮거나(냉난방 포기) 높은(노후 설비 고효율화 필요) 패턴은 에너지 빈곤의 프록시 신호로 활용 가능하다.

**③ 복지-에너지-AI 3중 결합**

복지(에너지바우처 법정 자격 요건) + 에너지 데이터(소비 패턴) + AI(추정 모델 + 챗봇)를 결합한 서비스는 현재 국내에 존재하지 않는다 [추정]. 각 영역은 독립적으로 존재하지만 이를 사용자 여정 전체로 통합한 사례가 없다.

#### AI 기술 구체 명세

**[A] 취약가구 자격 추정 모델**

- **모델 방식**: Random Forest + 규칙 기반 앙상블
  - 1단계(규칙 레이어): 에너지바우처 법정 자격 요건(기초생활수급자·차상위계층·장애인·65세 이상 노인·영유아 등) 을 결정 트리로 구현. 입력값이 법정 기준에 명확히 해당하면 확정 판정.
  - 2단계(ML 레이어): 지역별 전력 소비 패턴(계약종·월별 사용량·요금 분포), 에너지통계 소득분위별 에너지지출 분포, 기상(냉난방도일), 인구통계(고령화율)를 피처로 사용하는 Random Forest 분류 모델. 법정 기준 외 잠재적 취약가구 추정.
  - 출력: 자격 해당 확률(0~1) + 해당 근거 주요 피처 설명(SHAP 값 기반 설명 가능 AI)

- **학습 데이터**: 공개된 에너지바우처 수급 통계(지자체별 수급률)를 정답 레이블로 활용. 행정구역 단위(시·군·구) 집계 데이터로 지역 수준 모델 구축 → 개인정보 처리 최소화.

- **개인정보 보호**: 모델은 개인 식별 정보가 아닌 **지역 집계 통계 + 사용자 입력 익명 정보**를 처리. 개인의 전력 사용량 원시 데이터는 서버에 저장하지 않음.

**[B] 원클릭 신청 챗봇**

- **LLM 기반 대화 설계**: GPT-4o (또는 동급 오픈소스 LLM) 기반, 에너지바우처 신청 도메인 특화 RAG(Retrieval-Augmented Generation) 파이프라인 구성.
  - 지식 베이스: 에너지바우처 공식 안내문, 지자체별 신청 절차, 필요 서류 목록, FAQ를 벡터 DB(ChromaDB)에 임베딩.
  - 검색 증강: 사용자 질의 → 유사 문서 검색 → LLM이 문서 기반 답변 생성. 환각(Hallucination) 최소화.
  - 구조적 신청 플로: 자유 대화가 아닌 **단계별 구조적 폼 대화** 방식 (주소 입력 → 가구 유형 확인 → 자격 확인 → 서류 안내 → 신청 링크 제공). LLM은 각 단계의 안내 문구 생성과 예외 질문 처리에 특화.

- **API 래퍼를 넘는 독자 가치 (§AI 해자 논증)**:
  - 단순 LLM 호출이 아니라 **에너지바우처 도메인 지식 그래프(자격 요건 규칙 트리) + 지역별 신청 절차 데이터베이스 + 자격 추정 AI 모델**이 연결된 워크플로. LLM 교체(예: Claude → GPT) 시에도 도메인 지식 베이스·규칙 엔진·추정 모델은 그대로 유지된다.
  - 고령 친화 UX: 음성 입력·큰 글씨 모드·단계별 확인 화면은 범용 LLM 챗봇에 없는 이 서비스만의 레이어.
  - 신청 결과 추적: 신청 후 처리 현황 알림(카카오 알림톡 또는 문자) 기능은 LLM 독립적인 자체 백엔드 기능.

| 개인정보 보호 | 서비스 이용 시 개인 입력 최소화, 서버 저장 금지 설계 | 개인정보보호법·에너지법 준수 법률 자문 확보 [예정] |
| 경쟁·대체 위협 | 정부가 직접 자동 신청 시스템 구축 시 시장 위협 | 선제적 레퍼런스 구축 + 오픈소스 공개로 제도 내재화 기여 전략 |

## 경영혁신·창업학적 프레임워크

### 블루오션 전략 (Kim & Mauborgne)

기존 복지 IT 시장은 행정 처리 효율화(공급자 관점)에 집중되어 있다. 바우처 알리미는 **미발굴 수요자(사각지대 17만 가구)라는 비경쟁 블루오션**을 개척한다. 경쟁 없는 이유: 이 가구들은 현재 어떤 서비스도 사용하지 않기 때문이다(미사용자 시장).

| 블루오션 전략 요소 | 바우처 알리미 |
|:---|:---|
| 제거 (Eliminate) | 복잡한 신청 절차, 방문 상담 의존 |
| 감소 (Reduce) | 행정 처리 인력 투입, 신청 소요 시간 |
| 증가 (Raise) | 대상자 도달률, 신청 완료율, 데이터 기반 정확도 |
| 창조 (Create) | 능동적 취약계층 발굴, 에너지 데이터 복지 재목적화, 지자체 의사결정 지원 |

## 차별성·경쟁우위 (Moat) 및 구매동인 논증

### Why Us / Why Now

- **Why Now**: 에너지바우처 예산은 매년 확대되나 미수급 가구는 여전히 17만. 2026년 산업부가 에너지 취약계층 지원 강화를 정책 과제로 제시. AI + 공공데이터 결합의 기술 준비도(TRL) 충분.
- **Why Us**: 에너지 공공데이터(산업부 데이터) + 복지 도메인 + AI 결합 역량의 조합. 공모전 수상으로 산업부 신뢰 기반 선제 구축.

### 구매동인 논증

**대상 고객: 기초지자체 복지 담당 공무원**

| 구매동인 요소 | 내용 |
|:---|:---|
| 구매동인 가설 | 담당 공무원의 핵심 KPI는 "관할 지역 복지 수급률"과 "민원 감소". 취약지역 히트맵은 예산 배분 근거 문서화에 직접 기여. **Must-have**: 수급률 미달 시 상위기관 감사 위험. |
| 가치 정량화 | 담당 1인이 수급 가구 1개 발굴·안내에 소요하는 시간 약 60분 [추정]. 서비스 도입 시 자동화 → 동일 시간에 3~5배 더 많은 가구 처리 가능 [추정]. |
| 반증·대안 위협 | 공무원 개인 인센티브 구조 특성상 새 도구 도입보다 현상 유지 선호 가능. 대응: 무상 파일럿 + 성과 데이터로 윗선 설득 지원. |

**대상 고객: 에너지 취약계층 가구 (B2C)**

| 구매동인 요소 | 내용 |
|:---|:---|
| 구매동인 가설 | 연 평균 20만원 바우처를 신청하지 않은 이유: 제도 무지 + 신청 절차 복잡성. 챗봇으로 장벽 제거 시 신청 완료 의향 높음. **Must-have**: 금전적 혜택 자체가 동인. |
| 가치 정량화 | 1가구당 연 평균 20만원 지원[^4]. 신청 1회 소요 시간: 기존 방문 약 2~3시간 → 서비스 이용 약 10~20분 [추정]. |
| 반증·대안 위협 | 고령층 스마트폰 미보유 가구. 대응: 오프라인 QR 안내 + 복지관 직원 보조 활용으로 보완. |

## 데이터 정직성 선언

본 제안서의 모든 공식 통계(에너지바우처 지원 대상·실지급 가구수, 평균 지원액 등)는 각주([^번호])로 출처를 명시하였다. 시장 규모·업무 시간 절감·단위경제성 등 공인 통계가 없는 항목은 **[추정]**으로 표기하고, 근거로 삼은 가정을 함께 기재하였다. 검증된 외부 수치와 내부 추정값을 혼동하지 않았으며, 존재하지 않는 출처는 인용하지 않았다.
