# 산단 헬스맵 — 산단 활력·공실·노후화 분석 대시보드

- 전국 국가산업단지의 가동률·공실·노후화·생산·고용 지표를 단지별로 집계·시각화하고, AI 기반 활력지수 산정 및 쇠퇴 조기경보를 제공하는 공공 대시보드
- 정책 입안자(산업부·지자체)와 입주희망 기업이 단지 건강 상태를 한눈에 파악하고 선제적 지원·의사결정에 즉시 활용할 수 있게 한다
- **핵심 기술·서비스·정보 명칭**: 산단 활력지수(IVI, Industrial Vitality Index) / 쇠퇴 조기경보 모형 / 단지별 멀티지표 히트맵

## 1. 아이디어 기획 핵심내용 (구체성, 우수성)

### 1.1 무엇을 만드는가

**산단 헬스맵(KICOX 공공데이터 기반 국가산단 활력 대시보드)**은 한국산업단지공단(KICOX)이 공개하는 3종 공공데이터를 파이프라인으로 연결하여, **전국 국가산단 43개 단지**[^1]의 활력 상태를 실시간 지표로 시각화하는 웹 기반 공공 서비스다.

핵심 화면 구성은 다음과 같다.

- **헬스맵 (지도 레이어)**: 전국 지도 위 단지별 활력지수(IVI) 색조 히트맵. 단지 클릭 시 드릴다운 패널 전환
- **단지 카드**: 가동률·공실 면적·노후화 연령·생산액·수출액·고용인원을 KPI 카드로 표시
- **IVI 추이 차트**: 분기별 활력지수 시계열 그래프 + 임계선(경보 70 / 위기 60) 표시
- **조기경보 알림**: IVI 2분기 연속 하락 단지 자동 발굴 → 정책 담당자 대시보드 배너 표시
- **비교 분석**: 복수 단지 지표 병렬 비교 테이블

### 1.2 우수성 — 왜 지금 필요한가

국가산단 50인 미만 사업체 가동률은 2025년 기준 **69.6%로 70% 임계선 붕괴**[^2], 자진폐업은 전년 대비 **+15.4%** 증가했다[^3]. 노후산단(조성 후 30년 이상) 비중은 2030년 **50% 초과** 전망이다[^4]. 그러나 현재 산단 데이터는 KICOX 포털에 분기별 정적 Excel 형태로 존재할 뿐, **단지별 종합 건강 상태를 한눈에 보여주는 서비스는 없다**. 정책 입안자는 위기 단지를 사후에 파악하고, 입주 희망 기업은 단지 선택에 불필요한 조사 비용을 지불한다.

### 1.3 구현 방법 개요 (구체성)

| 계층 | 기술 선택 | 이유 |
|:---|:---|:---|
| 데이터 수집 | Python requests + KICOX CSV 정기 다운로드 (분기) | 공공데이터포털 파일 형식 대응 |
| 전처리·적재 | pandas 정규화 → DuckDB 경량 OLAP | 서버리스 배포 가능, SQL 지원 |
| IVI 산정 | 가중선형합 + 시계열 이상탐지 (Isolation Forest) | 해석 가능성 + 이상치 탐지 병행 |
| 조기경보 | 2분기 연속 IVI 하락 + Z-score < -1.5 이중 조건 | 오경보 억제 |
| 프론트엔드 | React + Leaflet.js (지도) + Recharts (차트) | 경량·오픈소스 |
| 배포 | Vercel 정적 + GitHub Actions 분기 데이터 갱신 | 무중단·저비용 |

## 2. 아이디어 구상 및 제안배경 (활용적정성)

### 2.1 문제 현황

**활용분야**: 산업정책(국가산단 지정·지원), 지역경제 활성화, 기업 입지 의사결정, 부동산·산단 투자 분석

**활용빈도**: 산업부·KICOX는 분기 1회 산단 통계를 갱신한다. 본 서비스는 데이터 갱신 주기에 맞춰 **분기 자동 업데이트**하며, 대시보드 접속은 정책 담당자·기업 관계자가 수시(일 단위) 조회한다.

**활용범위**: 전국 국가산단 43개[^1] 및 일반·도시첨단 산단을 포함 시 **1,282개 단지**[^5]로 확장 가능. 수혜자는 (1) 산업부·지자체 정책 담당자, (2) 입주 희망 제조기업, (3) 산단 내 공실 임대·매각 부동산 관계자, (4) ESG·지역재생 연구자.

**중요성**: 산업단지는 국내 제조업 생산의 **약 67%**, 수출의 **약 74%**를 담당[^6]하는 핵심 인프라다. 가동률 저하·공실 증가는 단순한 부동산 문제가 아니라 지역 고용·수출 경쟁력 손실을 의미한다. 국가산단 50인 미만 가동률이 **69.6%**(2025)[^2]로 하락한 현 상황에서 조기경보 체계는 더 이상 선택이 아니다.

### 2.2 구상 배경

산업부 공공데이터포털에는 KICOX가 제공하는 산단 입주업체 현황, 전국산업단지통계, 가동률·생산·수출·고용 동향 데이터가 분기 단위로 공개되어 있다. 그러나 이 데이터는 (1) **단지별로 흩어진 Excel 파일** 형태이고, (2) **연결된 분석 레이어 없이** 원시 수치만 제공되며, (3) **지도 시각화·조기경보·비교 분석**을 위한 어떤 프론트엔드도 존재하지 않는다.

결과적으로 정책 담당자는 위기 단지를 수동 집계 후 보고 체계를 통해 사후 인지하고, 입주 희망 기업은 단지 선택 시 직접 포털을 순회하며 수치를 비교한다. 본 아이디어는 이 공백을 "공공데이터의 결합·시각화·AI 분석"으로 채운다.

### 2.3 경영혁신·창업학적 프레임워크

**Christensen 파괴적 혁신(Disruptive Innovation) 관점**: 현재 산단 데이터 분석은 정부 연구기관·컨설팅펌이 고가 용역으로 수행하는 "고성능 고비용" 시장이다. 산단 헬스맵은 공공데이터를 기반으로 **무료·자동화**된 분석을 제공함으로써 이 시장 하단(정책 담당자의 일상적 모니터링 필요)을 파고드는 파괴적 혁신이다.

**JTBD(Jobs To Be Done)**: 정책 담당자의 핵심 Job은 "위기 단지를 조기에 발견해 선제 지원 예산을 편성하는 것"이고, 입주 기업의 Job은 "최소 조사 비용으로 최적 입지를 선택하는 것"이다. 기존 정적 Excel은 두 Job을 모두 불완전하게 충족한다. 헬스맵은 이 두 Job을 동시에 충족하는 멀티사이드 플랫폼이다.

**Porter 5 Forces — Why Now**: CBAM(탄소국경조정) 2026 시행, 수도권 제조업 이탈 심화, 노후산단 개선 촉진 특별법 논의 등 **구조적 규제·정책 전환**이 집중되는 시점이다. 데이터 수요가 급증하지만 공급(통합 분석 서비스)이 부재한 Blue Ocean이다.

**활용 내용 설명**

- **데이터셋 1 (입주업체 현황)**: 단지별 입주 업체 수, 가동 업체 수, 업종 분포를 제공한다. IVI 산정 시 '업체 가동률(가동 수 / 입주 수)' 지표로 활용하고, 업종 집중도(HHI)를 계산하여 공급망 취약성 보정 지표로 사용한다.
- **데이터셋 2 (전국산업단지현황통계)**: 단지별 조성연도·면적·분양면적·계약면적·공실면적을 제공한다. '노후화 연령(2026 - 조성연도)'과 '공실률(공실 면적 / 분양 면적)'을 계산하는 데 사용한다.
- **데이터셋 3 (가동률·생산수출고용 동향)**: 분기별 단지 가동률, 생산액, 수출액, 고용인원 추이를 제공한다. IVI의 핵심 4개 분기 시계열을 구성하며 조기경보 판단의 기초 데이터다.

### ④ 창의성·독창성

산단 헬스맵의 독창성은 **"공공 원자료의 파이프라인화 + AI 지수 산정 + 조기경보의 3단 구조"**에 있다. 기존에 각각 존재하는 요소(KICOX 통계 공개, 지수 산정 방법론, 웹 대시보드)를 **처음으로 단일 무료 공공 서비스로 통합**한다는 점이 독창적이다.

특히 **산단 활력지수(IVI)**는 기존 단편적 수치(가동률 단독)를 대체하는 **복합 진단 지표**로, 이를 공공 웹 서비스로 무료 제공하는 사례는 현재까지 없다(국책연구원의 유사 분석은 용역 보고서 형태로만 존재). 13회 수상작인 '자연어 데이터 분석(shannon)'이 LLM 기반 일반 데이터 질의에 초점을 맞춘 것과 달리, 산단 헬스맵은 **산단 도메인 특화 지수·경보 체계**를 구축한다는 점에서 기능적으로 중복되지 않는다.

### ⑤ 개요·구현기술·서비스방법 (AI 방식 구체화)

**구현 기술 스택**

| 계층 | 기술 | 세부 사항 |
|:---|:---|:---|

**AI 방식 구체 설명 (가점 항목)**

**(a) IVI 가중선형합 모형**

IVI는 다음 5개 지표의 가중합(0~100 스케일)으로 산정한다.

```
IVI = w1 × 가동률 + w2 × (1 - 공실률) + w3 × 생산액_증감 
      + w4 × 고용_증감 + w5 × (1 - 노후화_계수)
```

초기 가중치(w1~w5)는 국토연구원·산업연구원 선행 연구에서 전문가 설문 기반으로 도출된 중요도 순위를 참조하여 설정하며[^7], 분기 데이터 축적 후 **OLS 회귀로 가중치를 데이터 기반 업데이트**하는 구조다. [추정: 초기 가중치 w1=0.35, w2=0.25, w3=0.20, w4=0.15, w5=0.05는 설계 초안으로, 실제 데이터 검증 후 조정됨]

**(b) Isolation Forest 이상치 탐지**

분기별 5개 지표 벡터를 Isolation Forest에 투입하여 **정상 범위를 이탈한 단지를 자동 탐지**한다. 이 모델은 라벨 없이 학습하는 비지도학습이므로 "위기 사례" 레이블 없이도 이상 패턴을 발굴할 수 있다. 탐지 결과는 IVI 조기경보의 보조 신호로 활용한다.

**(c) ARIMA 시계열 예측**

단지별 IVI 시계열(분기)에 ARIMA 모형을 적합하여 **향후 3분기 IVI 예측값과 95% 신뢰구간**을 제공한다. 예측값이 경보 임계선(70) 이하로 하향 돌파할 것으로 예상되는 단지를 "예비 경보" 단지로 사전 표시한다.

**AI 해자 논증 (API 래퍼 아님)**

본 서비스의 AI는 ChatGPT·외부 LLM API 호출이 아니라, **KICOX 도메인 데이터 위에 구축된 자체 분석 파이프라인**이다. 핵심 독자 자산은 다음과 같다.

- **도메인 특화 지수 설계**: IVI는 산단 행정·산업 도메인 지식(노후화 연령·업종집중도 보정)이 반영된 자체 구조다. LLM이 이를 대체할 수 없다.
- **분기 누적 시계열 데이터베이스**: 공공데이터를 수집·정제·적재한 DuckDB가 핵심 자산이다. 시간이 지날수록 ARIMA·회귀 정확도가 높아지는 **데이터 축적 해자**가 형성된다.
- **모델 교체가능성 전제**: IVI 산정 수식과 조기경보 규칙은 기반 모델(ARIMA→Prophet 등)이 바뀌어도 유지된다. 분석 파이프라인 자체가 해자이며, 특정 AI 라이브러리 의존도는 낮다.

**서비스 제공 방법**

- **채널**: 공공 웹 대시보드 (무료 공개). 산업부·KICOX 링크 협력으로 공식 연계 추진.
- **사용자 흐름**: 지도 → 단지 클릭 → IVI 카드 확인 → 시계열 추이 조회 → 경보 목록 확인 → PDF 보고서 내보내기
- **데이터 갱신**: GitHub Actions로 KICOX 분기 데이터 공개 시 자동 수집·재산정·배포

- 분기 데이터 수집·정리 업무: [추정] 정책 담당자 1인 기준 분기 8~16시간 소요 → 연 32~64시간 절감
- 위기 단지 파악 리드타임: 현재 사후 보고 체계(분기+1~2개월) → 분기 종료 즉시 인지로 **약 2개월 선행 확보**[추정]
- 이 선행 시간이 실제 정책 예산 편성 타이밍에 영향을 준다면, 조기 지원으로 폐업 기업 1개 감소 = 지역 고용 5~50인 유지 가치[추정]

**③ 외부 근거**

- 국가산단 가동률 69.6% (2025) 통계: KICOX 산업동향[^2]
- 노후산단 2030 50% 전망: 산업연구원 보고서[^4]
- 자진폐업 +15.4% 증가: KICOX 입주업체 현황 분기 데이터[^3]

**④ 반증·대안 위협 직시**

- **위협 1**: 정책 담당자가 "KICOX에서 직접 달라"고 요청 → 대응: KICOX 자체 포털에는 IVI 종합 지수·지도 시각화·조기경보가 없음. 데이터 공급자와 분석 서비스는 역할이 다름.
- **위협 2**: "정부 내부 시스템(DW)에서 이미 처리함" → 대응: 내부 시스템은 공개되지 않으므로 외부 기업·연구자가 활용 불가. 헬스맵은 **공공 공개** 서비스로 포지셔닝.
- **위협 3**: 무료 서비스 → 지속가능성 의문 → 대응: Phase 2 B2G SaaS 전환으로 수익화, 초기 GitHub Actions 자동화로 운영비 최소화.

### 4.4 사회 파급효과 (정량 기대효과)

**표 2.** 정량 기대효과 목표

| 지표 | 현황 | 목표 (서비스 도입 후 3년) | 산출 근거 |
|:---|:---|:---|:---|
| 위기 단지 조기 발견 리드타임 | 사후 1~2개월 | 당분기 내 즉시 발견 | 자동 경보 체계 |
| 정책 담당자 분기 데이터 집계 시간 | [추정] 8~16시간/분기 | 1시간 미만 | 파이프라인 자동화 |
| 기업 입지 조사 기간 | [추정] 2~4주 | 1일 이내 | 단지 비교 대시보드 |
| 조기경보 대상 단지 수 (연간 발굴) | 없음 (수동 파악) | 연 5~10개 단지 선제 지원 | IVI 임계선 기준 [추정] |
| 서비스 이용 기관 수 (3년 목표) | 0 | 50개 기관 (지자체·관리기관) | GTM Phase 2 목표 [추정] |

> [추정] 표 내 기대효과 수치는 유사 공공 대시보드 도입 사례·KICOX 데이터 갱신 주기·담당자 업무 인터뷰 부재 상황을 감안한 설계 추정치다. 검증된 외부 수치는 §3 및 `5_research/README.md`에 출처 표기.

**사회적 가치**

1. **지역 고용 안정**: 위기 단지 조기 발견 → 선제 지원 → 폐업 기업 감소 → 지역 고용 유지
2. **정책 효율성 향상**: 예산 배분 근거를 객관적 IVI 수치로 제시 → 행정 투명성 제고
3. **공공데이터 활용 가치 실현**: KICOX 분기 데이터가 현재 "공개는 되지만 활용 안 되는" 상태에서 실질적 서비스로 전환됨으로써 공공데이터 개방의 사회적 가치 실현
4. **산단 재생 정책 근거 강화**: 노후화·공실 데이터의 지도화로 산단 재생 특별법 실행 근거를 구체화
