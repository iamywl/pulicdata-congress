# 2_개발계획서 — 리콜 레이더 (작업 컨텍스트 메모)

> 발주처 제출용 아님. 에이전트가 세션 간 맥락(무엇을·어디까지·자체/외주·현재 상태)을 잃지 않기 위한 메모.

## 목표 한 줄
국가기술표준원 제품 안전인증·리콜 정보(실시간 API, 15116894)를 **스캔·개인화 푸시**로 전환한 소비자 리콜 알림 데모앱 v1 — 키 없이 mock 으로 단독 구동·시연.

## 기술 스택
- 단일 자체완결 HTML(`projects/recall-radar/v1.html`) — 빌드 불필요, 오프라인·단독 구동, 버전 파일 보존(§3.5).
- Vanilla JS + 내장 CSS, 카메라는 `getUserMedia`(없으면 수동), 저장은 localStorage.
- 외부 의존: CDN 차트/아이콘 최소(오프라인 폴백). API 키 부재 → mock 데이터.
