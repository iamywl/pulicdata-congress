# 3_과업지시서_v1 — 리콜 레이더 데모앱 v1

## 1. 과업의 개요

**과업의 목적**
1) 직접 목적: 「리콜 레이더」 소비자 리콜 알림 데모앱 v1(자체완결 HTML) 1식 제작 — 발표평가 시연용 PoC.
2) 제안([1_제안서.md](./1_제안서.md))과의 연결: 제안서가 주장한 "국가기술표준원 실시간 리콜 데이터(15116894)를 pull 검색 → 스캔·개인화 푸시로 전환"을 실제로 구현·시연해 *구체성·문제해결 가능성*(§2.1-E 발표 채점)을 증명한다.

**과업의 범위**
1) 수행 업무: mock 데이터 구성 / 스캔 UI / 리콜 판정 / 구매이력 매칭·푸시 시뮬 / 직구 KC 사전체크 / 피드·검색·필터·분포시각화 / 반응형 / 기능 100+.
2) 범위 외(Out of Scope): 실제 KATS API 키 연동(키 발급 후 v2), 실 푸시 발송(FCM 실연동), 백엔드 서버·회원가입(시제품 단계 로그인 불필요 §3.4).

## 2. 과업의 세부 내용
- **데이터(mock)**: KATS 제품 안전인증·리콜(15116894)·국외리콜(15040707) 응답 구조를 모사한 50건+ 리콜 레코드(제품명·모델·제조사·리콜사유·조치·일자·카테고리·위해등급). 실 연동 경로는 코드 주석에 명시.
- **핵심 화면(뷰 8+)**: ① 홈 대시보드(오늘의 리콜·내 위험) ② 스캔(카메라/바코드/수동) ③ 리콜 상세 ④ 내 제품(구매이력) ⑤ 알림센터(개인화 푸시) ⑥ 리콜 피드(검색·필터·정렬) ⑦ 직구 사전체크 ⑧ 통계/분포(카테고리·기간 차트).
- **실 동작 기능(토스트 mock 금지)**: 스캔→매칭, 제품 등록→localStorage 지속, 리콜 매칭 알림 생성, 필터·검색·정렬, 카테고리 드릴다운, 위해등급 배지, KC 코드 사전체크 룰, CSV 내보내기, 즐겨찾기, 다크모드 등 → **누적 100+ 기능**.
- **반응형**: 모바일(390px) 바텀탭바·스캔 우선, PC(1280px) 사이드바·대시보드. 가로 overflow 0.
- **품질·성능**: 키 없이 단독 구동, 새로고침 후 상태 유지, 한글 깨짐 0, pageerror 0.
