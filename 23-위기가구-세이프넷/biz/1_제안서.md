# 위기가구 세이프넷 — 전기·가스 사용중단 감지로 고독사·위기가구 조기발굴

## 아이디어 간략 개요

전기·가스 사용량이 급감하거나 장기간 0에 수렴하는 가구가 밀집한 행정동은 채무불이행·경제위기·건강이상·고독사 직전 상태에 처한 취약 가구가 증가했다는 신호다. 본 서비스는 한국전력공사 계약종별 전력사용량(데이터셋 ID 15101360)과 한국가스공사 용도별 월 공급량(15129906)을 집계 단위로 분석해 이상 감소 시그널을 행정동 단위로 포착하고, 다변량 AI 위기 스코어링을 통해 지자체 복지담당자가 선제적으로 방문·개입할 수 있도록 우선순위 알림을 제공하는 공공 연계형 조기발굴 플랫폼이다.

**핵심 기술·서비스·정보 명칭**: 에너지 이상감지 기반 위기가구 조기발굴 플랫폼  
(Energy Anomaly-Driven Crisis Household Early-Detection, **EACHED**)

---

## 1. 아이디어 기획 핵심내용 (구체성, 우수성)

### 1.1 무엇을 만드는가

**EACHED**는 산업통상자원부 공공 에너지 데이터 두 스트림을 결합해, 가구 수 대비 에너지 소비가 평년·계절 대비 임계치 이하로 급감하거나 복수의 월에 걸쳐 0에 수렴할 때 이를 지역 위기 신호로 전환하는 시스템이다. 세 모듈로 구성된다.

**표 1.** EACHED 구성 모듈

| 모듈 | 기능 | 산출물 |
|:---|:---|:---|
| **① 이상감지 엔진** | 계약종·행정구역·용도별 시계열 이상탐지 (Isolation Forest / LSTM Autoencoder) | 행정동·가구 유형별 이상 지수 (0–1) |
| **② 위기 스코어링 AI** | 계절·기온·인구 구조를 보정한 다변량 위기 점수 (LightGBM + 지자체 피드백 루프) | 행정동별 위기 스코어 (0–100) + XAI 설명 |
| **③ 지자체 알림 대시보드** | 스코어 임계치(기본값 75) 초과 시 담당 복지사에게 모바일 푸시 + 방문 우선순위 목록 | 대시보드 + 방문 태스크 큐 |

### 1.2 왜 지금 이 방식인가 — 핵심 우수성 세 가지

**① 집계 공개 데이터의 재발견**: 가구 원천(개별 고객 단전·단가스 기록)은 비공개이나, 계약종별 × 행정구역별 사용량 집계의 시계열 이상은 해당 지역 내 에너지 미사용 가구 비중 증가와 상관된다. 이미 개방된 데이터를 복지 렌즈로 재해석하는 방식이므로 추가 데이터 확보 비용이 없다.

**② 개인정보 이슈 회피**: 집계 공개 데이터만 활용함으로써 개인정보보호법상 이슈를 구조적으로 회피한다. 가구 특정은 지자체 담당자가 현장 방문으로 수행하는 역할 분리 구조다.

**③ AI 설명가능성(XAI)으로 담당자 신뢰 확보**: SHAP 기반 XAI 레이어가 "이 행정동이 고위험인 이유 Top 3"를 자연어로 제공해, 담당자가 알림을 맹목적으로 따르지 않고 판단 근거를 이해하며 행동한다.

## 2. 아이디어 구상 및 제안배경 (활용적정성)

### 2.1 해소하는 사회문제 — 고독사·복지 사각지대의 규모와 구조

#### 고독사의 현황과 증가 추이

고독사는 혼자 사망한 후 상당 기간 방치되는 죽음이다. 보건복지부 「2023년 고독사 실태조사」에 따르면 2023년 고독사 사망자는 **3,661명**으로, 2019년 2,699명 대비 **35.6% 증가**했다[^1]. 연평균 증가율은 약 7.9%로, 이 추세가 유지되면 2028년에는 연간 5,000명을 초과할 것으로 추산된다[추정]. 연령별로는 50대가 가장 많고(약 32%), 60대 이상을 합산하면 전체의 55% 이상이다[^1]. 성별로는 남성이 약 78%를 차지한다[^1].

지역별로도 고독사는 심화되고 있다. 인구 과소 지역에서는 이웃 신고가 더 늦어지고, 중소도시·농촌의 고독사 비중이 증가하는 추세가 한국보건사회연구원 연구에서 확인된다[^2]. 서울·수도권이 절대 건수는 많으나, 인구 10만 명당 고독사율은 지방 중소도시가 높다[^2].

#### 복지 사각지대 — 에너지 빈곤과의 교차

에너지바우처 지원 대상 130.7만 가구 중 실지급을 받은 가구는 113.6만으로, 약 **17.1만 가구**가 지원받지 못했다[^3][^4]. 이들 중 상당수는 신청 절차 인지 부족, 주소 불일치, 정보 접근성 부족으로 인한 행정 사각지대에 속한다. 에너지 빈곤 가구(소득의 10% 이상을 에너지 비용에 지출하는 가구)는 국내 약 128만 가구로 추정된다[^3].

에너지 빈곤 가구와 고독사 고위험군은 인구 집단 면에서 크게 겹친다 — 1인 노인 가구, 저소득 만성질환자, 탈수급 경계층이 공통적이다. 에너지 요금을 납부하지 못해 공급이 중단되거나, 요금 부담으로 스스로 사용을 극도로 줄이는 가구는 "에너지 사용 소멸 → 생활 단절 → 고립 심화"의 경로를 밟는다[^5].

## 3. 아이디어 세부내용

### 3.1 ① 활용한/활용할 산업통상자원부 공공데이터 (탈락요건 충족)

본 아이디어의 핵심 데이터는 산업통상자원부 산하기관인 한국전력공사(KEPCO)와 한국가스공사(KOGAS)의 공공데이터로서, data.go.kr에 개방되어 있다.

**표 4.** 핵심 산업통상자원부 공공데이터셋

| 구분 | 데이터셋명 | 데이터셋 ID | 제공기관 | data.go.kr URL | 활용 방식 |
|:---|:---|:---:|:---|:---|:---|
| **핵심 1** | 계약종별 전력사용량 | **15101360** | 한국전력공사 (KEPCO) | https://www.data.go.kr/data/15101360/openapi.do | 계약종(주거용·일반용 등)별 고객 호수·사용량·요금을 기간별로 수집, 행정구역 교차 후 시계열 이상탐지 입력 |
| **핵심 2** | 용도별 월 공급량 | **15129906** | 한국가스공사 (KOGAS) | https://www.data.go.kr/data/15129906/fileData.do | 도시가스 용도별(가정용·업무용 등) 월별 공급량 변화 분석, 가정용 공급량 급감 패턴 감지 |

**데이터 활용 전략**: 두 데이터셋 모두 집계 수준 공개 데이터로 개인정보를 포함하지 않는다. 계약종별·행정구역별 집계값의 시계열 변화를 분석함으로써, 가구 수 대비 사용량 비율 변화(= 가구당 평균 사용량 이상 감소)를 포착한다. 이 집계 이상값이 높은 행정동은 에너지 미사용 가구가 증가했음을 시사하므로, 지자체 담당자가 해당 동 내 취약 가구를 우선 방문하는 트리거가 된다.

> **투명성 고지**: 가구 원천 데이터(개별 고객 단전·단가스 기록)는 비공개이므로, 본 아이디어는 집계 공개 데이터의 이상 패턴을 지역 위기 신호로 활용하는 **간접 추론 방식**을 채택한다. 지자체·한전·가스사 3자 협업을 통해 가구 단위 확인은 현장 방문으로 이루어진다. 데모 앱에서 활용하는 에너지 이상 수치는 공개 집계 데이터를 기반으로 하며, 개인 식별 정보는 포함하지 않는다.

| **담당자 도구** | 엑셀·메모·전화 — 정보 파편화 | 행정동 위기 지도 + 방문 우선순위 자동 산출 |
| **AI 설명가능성** | 없음 | SHAP 기반 XAI — "왜 이 행정동이 고위험인지" 자연어 제시 |
| **데이터 피드백 루프** | 없음 | 방문 결과 → 모델 재학습 → 정밀도 향상 |
| **13회 수상작 차별** | — | 식품통관도우미·자연어 분석·재생에너지 기상보정과 도메인·데이터·타깃 완전 상이 |

### 3.4 ④ 창의성·독창성

**에너지 소비 데이터의 결핍 패턴을 복지 위기 신호로 전환한다**는 발상이 핵심이다.

기존에는 에너지 데이터를 요금 정산·수급 예측·탄소 계산에만 활용해 왔다. 본 아이디어는 이 데이터의 **결핍(소비 급감·소멸) 패턴**을 사회 위기 신호로 재해석한다. "전기가 끊겼다"는 사실이 위기를 뜻할 수 있음을 알고 있으면서도, 현재 시스템은 그 신호를 복지 체계와 연결하지 못하고 있다. 공개 집계 데이터만으로 가능한 이 추론 경로는 개인정보 보호를 훼손하지 않으면서도 사각지대를 줄일 수 있다.

또한 복지·에너지 두 부처 데이터를 연결하는 **부처 간 데이터 가교** 역할을 한다. 산업통상자원부 에너지 데이터가 복지부 돌봄 행정에 연결되는 부처 간 협업 모델로서, 부처 칸막이 데이터 활용의 선례가 된다.

#### AI 구현 방식 (3단계 파이프라인)

**1단계 — 시계열 이상탐지 (비지도 학습)**

- **알고리즘**: Isolation Forest(소규모 시계열·이상 빈도 낮을 때 강인) 또는 LSTM Autoencoder(패턴 복잡성이 높을 때 전환)
- **입력 피처 (6종)**:
  1. 전월 대비 주거용 전력사용량 변화율 (%)
  2. 전년 동월 대비 변화율 (%)
  3. 도시가스 가정용 월 공급량 변화율 (%)
  4. 냉난방도일(HDD/CDD) 보정값
  5. 행정동 내 1인 가구 비율 (%)
  6. 행정동 내 65세 이상 인구 비율 (%)
- **출력**: 행정동별 이상도 점수 (0–1), 이상 원인 상위 피처

**2단계 — 위기 스코어링 (지도 학습·피드백 루프)**

- **알고리즘**: LightGBM 기반 분류 모델
- **초기 부트스트랩**: 전문가 정의 규칙으로 시작
  - 3개월 연속 주거용 전력사용량 전년 대비 ≤50% 면서 1인 고령 가구 비율 ≥20%인 행정동 → 초기 고위험 라벨
  - 가스 가정용 공급량 3개월 연속 0 근방(전년 대비 ≤10%) → 추가 가중
- **피드백 루프**: 지자체 담당자가 방문 결과(위기 확인/정상)를 앱에 입력 → 라벨로 축적 → 분기 1회 재학습 → 오탐률 점진 감소
- **이것이 단순 API 래퍼와 다른 독자 데이터 자산**: 사용 결과가 쌓일수록 정밀도가 향상되는 데이터 네트워크 효과

**3단계 — 설명 가능 AI (XAI)**

- SHAP(SHapley Additive exPlanations) 값으로 각 행정동의 위기 점수 기여 요인 Top 3를 자연어로 담당자에게 제시
- 예시 출력: "OO동 위기 스코어 82 — ① 주거용 전력사용량 전월 대비 −61% (기여 38%), ② 1인 노령 가구 비율 34% (기여 27%), ③ 가스 가정용 전년 대비 −43% (기여 21%)"

#### API 래퍼가 아닌 독자 해자 논증

**표 7.** EACHED의 독자 해자 — Why Not A Wrapper

| 해자 유형 | 내용 |
|:---|:---|
| **독자 데이터 자산** | 지자체 현장 방문 결과(위기 확인 여부)를 피드백 라벨로 축적 → 공개 모델로 복제 불가능한 도메인 특화 훈련 데이터. 도입 자치단체가 늘수록 전국 피드백 DB 성장 → 정밀도 향상 가속 |
| **버티컬 워크플로 통합** | 이상탐지 → 스코어링 → 방문 우선순위 → 결과 피드백까지 업무 전 주기 통합 파이프라인. 단발 API 호출과 달리 담당자의 일상 업무 루틴에 깊이 통합 |
| **도메인 보정 파라미터** | 계절·지역·인구 구조 보정 가중치 — 지자체별 특성에 맞춰 튜닝된 파라미터는 서비스에 고착(모델 교체 후에도 잔존) |
| **모델 교체가능성 전제** | 기반 이상탐지 알고리즘이 교체되어도 지자체 피드백 라벨 DB·도메인 보정 파라미터·XAI 설명 규칙은 서비스에 남음. 기반 모델 상품화에 의존하지 않음 |

#### 서비스 운영 방법 (6단계)

1. **데이터 수집**: 월 1회 data.go.kr API·파일에서 KEPCO 전력사용량(ID 15101360)·KOGAS 공급량(ID 15129906) 자동 수집 + 기상청 HDD/CDD 자동 취합
2. **이상탐지 실행**: 수집 후 익일, 행정동별 이상도 점수 산출 (배치 처리, 전국 약 3,500개 행정동 대상)
3. **위기 스코어 산출**: 이상도·보정 변수·인구 구조를 통합해 위기 스코어(0–100) + XAI 설명 생성
4. **스코어 알림**: 임계치(기본값 75 이상) 초과 행정동 발생 시 해당 읍·면·동 복지팀 담당자에게 앱 푸시 알림
5. **현장 방문 및 피드백**: 담당자 방문 결과(위기 확인 여부)를 앱에서 입력 → 모델 재학습 트리거 (분기 1회)
6. **월보 자동 생성**: 행정동별 위기 발굴 현황·개입 건수·예방 추정 효과를 자동 집계해 시·군·구 복지부서에 보고

#### 경영혁신·창업학적 프레임워크

**Clayton Christensen의 파괴적 혁신(Disruptive Innovation)** 관점에서, 기존 고독사 예방 서비스는 전담 인력 방문 + 주관적 신고 중심의 고비용·저효율 구조다. EACHED는 공공 에너지 데이터라는 이미 존재하지만 활용되지 않는 자원을 결합해, 기존 복지 인력이 더 적은 시간에 더 많은 위기가구를 찾을 수 있는 낮은 비용의 대안 경로를 만든다. 기존 "고성능 인력 집약형" 방식의 아래를 파고드는 로우엔드 파괴에 해당한다.

**Jobs To Be Done(JTBD)**: 지자체 복지담당자의 핵심 과업은 "위기가구를 위기가 심화되기 전에 찾아 개입하는 것"이다. 이 과업은 현재 담당 가구 과다·방문 주기 길어 불충분하게 수행되며, EACHED는 이 JTBD를 데이터 기반으로 해결한다.

**Osterwalder BMC 주요 블록**: 핵심 자원 = 에너지 집계 데이터 + AI 모델 + 지자체 피드백 DB / 핵심 채널 = 공공 입찰·시범 사업 / 수익원 = SaaS 구독 / 비용 = 서버·인건비.

**린 스타트업(Eric Ries)**: 시범 5~10개 자치단체 MVP → 방문 효율 데이터 실측 → 오탐률 측정 → 모델 개선 반복의 Build–Measure–Learn 사이클.

### 4.4 실현가능성

#### 기술 실현 가능성

- data.go.kr 공개 API·파일은 즉시 수집 가능 (별도 협약 불필요)
- Isolation Forest·LSTM Autoencoder·LightGBM 모두 오픈소스 라이브러리(scikit-learn, PyTorch, LightGBM)로 구현 가능
- 초기 모델은 규칙 기반 부트스트랩으로 데이터 축적 전에도 서비스 가능
- SHAP 라이브러리는 오픈소스로 즉시 적용 가능

#### 제도적 실현 가능성

- **고독사예방법(2021.04 시행)**: 지자체에 고독사 예방 계획 수립·시행 의무 부여 → EACHED가 이행 수단으로 기능
- **개인정보보호**: 집계 공개 데이터 활용으로 개인 식별 없이 운영 가능
- **행정 데이터 연계**: 사회보장기본법·사회보장정보원 통합 연계망(행복e음)과 중장기 연동 경로 수립 가능

## 데이터 정직성 선언

본 제안서의 모든 통계·인용은 아래 각주로 출처를 명시하였다. `[추정]` 표기가 붙은 수치는 공식 데이터가 없어 산출한 추정치이며, 공식 수치와 혼용하지 않았다. 데이터셋 ID는 실제 data.go.kr에 존재하는 검증된 ID만 사용하였으며, 새로운 ID를 날조하지 않았다. 없는 출처를 날조하거나 검증되지 않은 수치를 사실로 표현하지 않았다.
