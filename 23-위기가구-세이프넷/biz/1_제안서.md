# 위기가구 세이프넷 — 전기·가스 사용중단 감지로 고독사·위기가구 조기발굴

## 아이디어 간략 개요

전기·가스 사용량이 급감하거나 장기간 0에 가까워진 가구는 채무불이행·경제위기·건강이상·고독사 직전 상태일 가능성이 높다. 본 서비스는 한국전력 계약종별 전력사용량·한국가스공사 용도별 월 공급량을 집계 단위로 분석해 이상 감소 시그널을 지역·동(洞)·가구 유형 단위로 포착하고, AI 위기 스코어링을 통해 지자체 돌봄 담당자가 선제적으로 방문·개입할 수 있도록 알림을 제공하는 공공 연계형 조기발굴 플랫폼이다.

**핵심 기술·서비스·정보 명칭**: 에너지 이상감지 기반 위기가구 조기발굴 플랫폼 (Energy Anomaly-Driven Crisis Household Early-Detection, EACHED)

## 1. 아이디어 기획 핵심내용 (구체성, 우수성)

### 1.1 무엇을 만드는가

**EACHED**는 두 가지 공공 에너지 데이터 스트림을 결합하여, 가구 단위 에너지 소비가 평년·계절 대비 임계치 이하로 급감하거나 수개월 연속 0에 수렴할 때 이를 위기 신호로 전환하는 시스템이다.

구체적으로는 다음 세 모듈로 구성된다.

| 모듈 | 기능 | 산출물 |
|:---|:---|:---|
| **① 이상감지 엔진** | 공개 집계 데이터(계약종·행정구역·용도별)의 시계열 이상탐지 | 행정동·가구 유형별 위기 이상 지수 |
| **② 위기 스코어링 AI** | 계절·날씨·지역 특성을 보정한 다변량 위기 점수 | 지역별·유형별 위기 스코어 (0~100) |
| **③ 지자체 알림 대시보드** | 스코어 임계치 초과 시 담당 사회복지사에게 알림·방문 요청 생성 | 대시보드 + 모바일 알림 |

### 1.2 왜 지금 이 방식인가

기존 위기가구 발굴은 (a) 이웃 신고, (b) 정기적 방문, (c) 공과금 체납 정보 행정 공유 세 경로에 의존한다. (a)와 (b)는 주기가 길고 인력 부족으로 포괄률이 낮으며, (c)는 한전·가스사→지자체 공유가 개인정보 이슈로 단편적이다. EACHED는 **집계 수준 공개 데이터를 활용**함으로써 개인정보 이슈를 우회하면서도, 행정동 단위 이상 신호를 포착해 현장 담당자의 방문 우선순위를 설정할 수 있게 한다.

### 1.3 핵심 우수성

- **공개 집계 데이터의 재발견**: 개별 가구 원천 데이터 없이도, 계약종별(주거용·일반용 등) × 행정구역별 사용량 집계의 시계열 이상은 그 지역 내 취약 가구 비중 급증과 상관된다.
- **AI 이상탐지 + 인과 설명**: 단순 임계치가 아니라 계절·기온·전년 동기 대비 다변량 보정 후 이상도를 산출해 오탐을 줄인다. 설명 가능 AI(Explainable AI) 방식을 채택해 담당자가 "왜 이 지역이 고위험인지" 인지할 수 있도록 한다.
- **지자체 실무 연계**: 대시보드는 읍·면·동 복지담당자가 직접 쓸 수 있는 업무 도구로 설계되며, 기존 복지행정시스템(주민등록·복지로)과의 연계 확장 경로를 열어 둔다.

## 2. 아이디어 구상 및 제안배경 (활용적정성)

### 2.1 해소하는 사회문제 — 고독사·복지 사각지대

#### 고독사의 현황과 규모

고독사는 혼자 사망한 후 상당 기간 방치되는 죽음으로, 2023년 기준 한국의 고독사 사망자는 3,661명[^1]으로 집계된다. 이 수치는 2019년 대비 약 35% 증가했으며[^1], 실제 수는 집계 범주 외 사례를 포함하면 더 클 것으로 추정된다[추정]. 연령별로는 50대가 가장 많으며(약 32%), 60대 이상을 합산하면 절반을 초과한다[^1].

지역 단위로도 고독사는 심화되고 있다. 2023년 기준 특별·광역시 외 중소도시와 농촌 지역에서 고독사 비중이 높아지는 추세가 확인된다[^2]. 인구 과소지역에서는 이웃 신고가 더욱 늦어지는 구조적 취약성이 있다.

#### 복지 사각지대 — 에너지 빈곤과의 교차

에너지바우처 지원 대상 130.7만 가구 중 실제 지급을 받은 가구는 113.6만으로, 약 **17만 가구**가 지원을 받지 못하고 있다[^3][^4]. 이들 중 상당수는 신청 절차를 모르거나, 주소 불일치, 정보 접근성 부족으로 인한 행정 사각지대에 놓인다.

에너지 빈곤 가구는 동시에 고독사 고위험군과 높은 중첩을 보인다. 1인 노인 가구, 저소득 만성질환자, 탈수급 경계층이 대표적이다. 전기·가스 요금을 내지 못해 공급이 중단되거나, 요금 부담으로 스스로 사용을 극도로 줄이는 가구는 "에너지 사용 소멸 → 생활 단절 → 고립 심화"의 경로를 밟는다[^5].

#### 현재 발견 체계의 한계 — '사후 발견' 문제

현행 위기가구 발굴 경로는 세 가지다.

| 경로 | 주기·방식 | 한계 |
|:---|:---|:---|
| 이웃 신고 (긴급복지지원 신청) | 비정기·수동 | 1인 고립 가구는 인지하는 이웃 자체가 없음 |
| 복지담당자 정기 방문 | 분기~연 1회 | 전국 사회복지사 1인당 담당 가구 평균 약 450가구[^6][추정] — 주기적 방문 불가 |
| 공과금 체납 정보 행정 공유 | 단편·사후 | 한전·가스사→지자체 정보 공유는 체납 후 단전·단가스 조치 시점(이미 위기 심화 후) |

세 경로 모두 위기가 이미 심화된 뒤에야 개입이 이루어지는 **사후 발견** 구조다. EACHED는 이 구조를 바꿔 **에너지 이상 신호 → 사전 개입**의 경로를 만든다.

### 2.2 활용분야·활용빈도·활용범위·중요성

| 요소 | 내용 |
|:---|:---|
| **활용분야** | 지자체 복지행정, 사회안전망 운영, 공공 에너지 데이터 재활용, 고독사 예방 정책 |
| **활용빈도** | 지자체 복지담당자: 주 1회 이상 대시보드 점검 / AI 스코어링: 월간 갱신(데이터 공개 주기 기준) → 점진적으로 주간 갱신 목표 |
| **활용범위** | 전국 229개 기초자치단체 읍·면·동 복지팀(광역 확장 가능), 1인 노인·저소득 가구 밀집 지역 우선 |
| **중요성** | 고독사 사망자 연 3,661명, 에너지 사각지대 약 17만 가구 — 조기 개입 1건이 고독사 1건 예방에 직결되므로 사회적 손실(사후 처리 비용 + 인적 손실) 절감 효과가 크다 |

## 3. 아이디어 세부내용

### 3.1 ① 활용한/활용할 산업통상자원부 공공데이터

본 아이디어의 핵심 데이터는 산업통상자원부 산하기관인 **한국전력공사(KEPCO)**와 **한국가스공사(KOGAS)**의 공공데이터로서, data.go.kr에 개방되어 있다. 탈락요건 충족을 위해 데이터셋명·기관·URL을 아래에 명시한다.

| 구분 | 데이터셋명 | 제공기관 | data.go.kr URL | 활용 방식 |
|:---|:---|:---|:---|:---|
| **핵심 1** | 계약종별 전력사용량 | 한국전력공사 (KEPCO) | https://www.data.go.kr/data/15101360/openapi.do | 계약종(주거용·일반용 등)별 고객 호수·사용량·요금을 기간별로 수집, 행정구역 교차 분석 후 시계열 이상탐지 입력 |
| **핵심 2** | 용도별 월 공급량 | 한국가스공사 (KOGAS) | https://www.data.go.kr/data/15129906/fileData.do | 도시가스 용도별(가정용·업무용 등) 월별 공급량 변화 분석, 가정용 공급량 급감 패턴 감지 |

**데이터 활용 전략**: 두 데이터셋 모두 **집계 수준 공개 데이터**로서 개인정보를 포함하지 않는다. 계약종별·행정구역별 집계값의 시계열 변화를 분석함으로써, 가구 수 대비 사용량 비율 변화(= 가구당 평균 사용량 이상 감소)를 포착한다. 이 집계 이상값이 높은 지역은 에너지 사용 중단 가구가 증가했음을 시사하므로, 지자체 담당자가 해당 행정동 내 취약 가구를 우선 방문하는 트리거가 된다.

> **참고**: 가구 원천 데이터(개별 고객 단전·단가스 기록)는 비공개이므로, 본 아이디어는 집계 공개 데이터의 이상 패턴을 지역 위기 신호로 활용하는 간접 추론 방식을 채택한다. 지자체·한전·가스사 3자 협업을 통해 개인정보보호법 준수 하에 가구 단위 확인은 현장 방문으로 이루어진다.

### 3.2 ② 타 기관·민간 데이터 (보조 결합)

| 데이터 | 기관/출처 | 활용 목적 |
|:---|:---|:---|
| 주민등록 인구통계 (행정구역별 1인 가구·노령 인구) | 행정안전부 | 에너지 이상 지역의 고위험 인구 구조 파악 |
| 기상청 기온·냉난방도일(HDD/CDD) | 기상청 | 계절적 에너지 변동 보정 (겨울 난방 감소를 위기 신호로 오탐하지 않도록) |
| 복지로 수급자 정보 (지자체 연계 시) | 보건복지부 (지자체 협업 전제) | 기존 수급자 미방문 기간 교차 확인 |
| 에너지바우처 지급 현황 | 한국에너지공단 (산업부 산하, 보조) | 바우처 미지급 가구 + 에너지 이상 지역 교차 — 이중 취약 가구 우선 발굴 |

### 3.3 ③ 기존 서비스 대비 차별성

**기존 위기가구 발굴·고독사 예방 서비스와의 비교**

| 비교 축 | 기존 서비스 | EACHED (본 아이디어) |
|:---|:---|:---|
| **발견 시점** | 사후(체납→단전 조치 후, 또는 이웃 신고 후) | 사전(에너지 사용 이상 감지 → 방문 트리거) |
| **데이터 기반** | 주관적 신고·정기 방문 의존 | 객관적 공공 에너지 데이터의 시계열 분석 |
| **지역 커버리지** | 이미 알려진 취약 가구 중심 | 행정동 전체를 스캔 — 미발굴 가구 포함 |
| **개인정보 부담** | 가구별 체납 정보 공유 시 법적 이슈 | 집계 공개 데이터 활용 → 개인정보 이슈 최소 |
| **AI 보정** | 없음(계절·날씨 보정 없어 오탐률 높음) | 기온·계절·지역 보정 다변량 AI 스코어링 |
| **담당자 도구** | 엑셀·메모·전화 — 정보 파편화 | 행정동 단위 위기 지도 + 방문 우선순위 자동 산출 |
| **13회 수상작 차별** | — | 식품통관도우미·자연어 분석·재생에너지 기상보정과 도메인·데이터·타깃 완전 상이 |

**13회 수상작과의 차별성 명시**: 제13회 수상작은 ① 식품 통관 도우미(무역·식품 안전), ② 자연어 데이터분석(챗봇·분석 플랫폼), ③ 재생에너지 기상보정(발전량 예측)으로, 복지·에너지 소비 이상 감지·사회 취약계층 조기발굴과는 도메인이 전혀 다르다.

### 3.4 ④ 창의성·독창성

**에너지 소비 데이터를 복지 위기 신호로 전환한다**는 발상이 핵심이다.

기존에는 에너지 데이터를 요금 정산·수급 예측·탄소 계산에만 활용해 왔다. 본 아이디어는 이 데이터의 **결핍(소비 급감·소멸) 패턴**을 사회 위기 신호로 재해석한다. "전기가 끊겼다"는 사실이 위기를 뜻할 수 있음을 알고 있으면서도, 현재 시스템은 그 신호를 복지 체계와 연결하지 못하고 있다. 공개 집계 데이터만으로 가능한 이 추론 경로는 개인정보 보호를 훼손하지 않으면서도 사각지대를 줄일 수 있는 독창적 접근이다.

또한 복지-에너지 두 부처 데이터를 연결하는 **부처 간 데이터 가교** 역할을 한다는 점에서, 단일 도메인 활용에 그치는 기존 서비스와 구별된다.

### 3.5 ⑤ 개요·구현기술·서비스 방법

**그림 1.** EACHED 시스템 데이터 흐름도

#### AI 구현 방식 (구체)

본 서비스의 AI는 단순 임계치 경보가 아니라 두 단계 학습 모델로 구성된다.

**1단계 — 시계열 이상탐지 (Unsupervised)**

- 알고리즘: **Isolation Forest** (소규모 시계열·이상 빈도 낮을 때 강인) 또는 **LSTM Autoencoder** (패턴 복잡성이 높을 때 적용 전환)
- 입력 피처: 전월 대비 전력사용량 변화율, 전년 동월 대비 변화율, 가스 공급량 변화율, 냉난방도일 보정값, 1인 가구 비율, 노령 인구 비율
- 출력: 행정동별 이상도 점수 (0~1)

**2단계 — 위기 스코어링 (Supervised, 지자체 피드백 루프)**

- 알고리즘: **LightGBM** 기반 분류 모델 (위기 확인 방문 결과를 라벨로 점진 학습)
- 초기: 전문가 정의 규칙(3개월 연속 전력사용량 전년 대비 50% 이하, 가스 공급 0 등)으로 부트스트랩
- 지자체 방문 결과(실제 위기/정상)를 피드백으로 받아 모델 정밀도 향상 — 이것이 **단순 API 래퍼와 다른 독자 데이터 자산**이다(사용 결과가 쌓일수록 오탐률이 줄어드는 데이터 네트워크 효과)

**3단계 — 설명 가능 AI (XAI)**

- SHAP(SHapley Additive exPlanations) 값으로 "이 행정동의 점수가 높은 이유"를 담당자에게 자연어로 제시 ("전월 대비 전력 사용량 -62%, 1인 노령 가구 비율 34% — 이상 집중")

#### API 래퍼가 아닌 독자 해자 논증

- **독자 데이터 자산**: 지자체 현장 방문 결과(위기 확인 여부)를 피드백 라벨로 축적 → 공개 모델로 복제 불가능한 도메인 특화 훈련 데이터
- **버티컬 워크플로 통합**: 이상탐지 → 스코어링 → 방문 우선순위 → 결과 피드백까지 전 과정이 하나의 파이프라인에 통합되어, 단발 API 호출과 달리 업무 전 주기에 개입
- **모델 교체가능성 전제**: 기반 이상탐지 알고리즘(Isolation Forest → LSTM → 타 모델)이 교체되어도, 지자체 피드백 라벨 DB와 도메인 보정 파라미터(계절·지역 가중치)는 서비스에 남는다

#### 서비스 방법

1. **데이터 수집**: 월 1회 data.go.kr API/파일에서 KEPCO 전력사용량·KOGAS 공급량 자동 수집
2. **이상탐지 실행**: 수집 후 익일, 행정동별 이상도 점수 산출 (배치 처리)
3. **스코어 알림**: 임계치(예: 이상도 0.75 이상) 초과 행정동이 발생하면 해당 읍·면·동 복지팀 담당자에게 앱 푸시 알림 발송
4. **대시보드 확인**: 담당자가 지역 위기 지도 + 이상 원인 설명 확인, 방문 가구 우선순위 결정
5. **현장 방문 및 피드백**: 방문 결과(위기 확인 여부)를 앱에서 입력 → 모델 재학습 트리거
6. **월보 자동 생성**: 행정동별 위기 발굴 현황·개입 건수·예방 추정 효과를 자동 집계해 시·군·구 복지부서에 보고

#### 경영혁신·창업학적 프레임워크

**Clayton Christensen의 파괴적 혁신(Disruptive Innovation) 관점**에서, 기존 고독사 예방 서비스는 "전담 인력 방문 + 주관적 신고" 중심의 고비용·저효율 구조다. EACHED는 공공 에너지 데이터라는 이미 존재하지만 활용되지 않는 자원을 결합해, 기존 복지 인력이 더 적은 시간에 더 많은 위기가구를 찾을 수 있는 낮은 비용의 대안 경로를 만든다. 이는 기존 "고성능 인력 집약형" 방식의 아래를 파고드는 로우엔드 파괴에 해당한다.

**Jobs To Be Done(JTBD) 관점**: 지자체 복지담당자의 핵심 과업은 "위기가구를 위기가 심화되기 전에 찾아 개입하는 것"이다. 현재 이 과업은 불충분하게 수행되고 있으며(담당 가구 과다·주기 길어), EACHED는 이 JTBD를 데이터 기반으로 해결한다.

### 4.3 실현가능성

#### 기술 실현 가능성

- data.go.kr 공개 API·파일은 즉시 수집 가능 (별도 협약 불필요)
- Isolation Forest·LSTM Autoencoder·LightGBM 모두 오픈소스 라이브러리(scikit-learn, TensorFlow/PyTorch, LightGBM)로 구현 가능
- 초기 모델은 규칙 기반 부트스트랩으로 데이터 축적 전에도 서비스 가능

#### 제도적 실현 가능성

- **고독사예방법(2021.04 시행)**: 지자체에 고독사 예방 계획 수립·시행 의무 부여 → EACHED가 이행 수단으로 기능
- **개인정보보호**: 집계 공개 데이터 활용으로 개인 식별 없이 운영 가능 (가구 원천 데이터는 지자체 자체 보유·활용 체계 안에서 현장 담당자가 접근)
- **행정 데이터 연계**: 사회보장기본법·사회보장정보원 통합 연계망(행복e음)과 장기 연동 로드맵 수립 가능
