# 버팀목 — 업종·지역 전력 둔화 기반 소상공인 폐업위기 조기경보·지원연계

> **아이디어 간략 개요 (3줄 이내)**
> 한국전력 산업분류별·계약종별 전력사용량 공공데이터를 업종·지역 단위로 집계·분석하여, 매출 둔화보다 먼저 나타나는 전력소비 이상 감소를 자동 포착하고, 위기 상권 소상공인에게 폐업 전 정부 지원사업을 선제 연계하는 조기경보·지원매칭 서비스다.
> 폐업은 통상 전력소비 둔화 이후 3~6개월 뒤 통계로 잡히므로[추정], 이 시차를 역이용해 가장 빠른 경기 선행지표인 전력데이터로 '사전 위기 신호'를 포착한다.
> 위기 상권 지도·소상공인 대상 지원연계 알림·유관기관(소상공인시장진흥공단·지자체) 대시보드를 3개 레이어로 제공한다.

**핵심 기술·서비스·정보 명칭**
- 전력기반 매출둔화 추정 모델 (전력소비 이상탐지 + 업종별 보정 알고리즘)
- 업종·지역 폐업위험 조기경보 스코어 (Power-Risk Score, PRS)
- 소상공인 지원사업 자동매칭 엔진
- 위기 상권 대시보드 (소진공·지자체 전용 / 소상공인 셀프 조회)

## 1. 아이디어 기획 핵심내용 (구체성, 우수성)

### 1.1 무엇을 만드는가 — 서비스 개요

**버팀목**은 한국전력(KEPCO)이 공개하는 산업분류별·계약종별 전력사용량 공공데이터를 핵심 원천으로, 업종(음식점·소매·숙박 등)과 행정구역(읍면동 단위[추정], 시군구 단위 확정) 교차로 월별 전력소비 추세를 추적한다. 전력소비가 해당 업종·지역 평년 대비 일정 임계값 이하로 떨어지면 '위기 상권'으로 플래그하고, 소상공인진흥공단·지자체·소상공인 본인에게 각각 맞춤 알림과 지원연계 정보를 제공한다.

### 1.2 핵심 구현 내용

**① 전력기반 위기 신호 포착**
- KEPCO 산업분류별 전력사용량(데이터셋 15101403)과 계약종별 전력사용량(15101360)을 월별 API로 수집한다.
- 업종 대분류(한국표준산업분류 A~Q)를 소매·음식·숙박·서비스 등 소상공인 밀집 업종군으로 필터링한다.
- 시군구 행정코드와 결합하여 '업종 × 지역' 매트릭스를 구성하고, 최근 12개월 이동평균 대비 직전 2개월 편차를 Z-score로 환산한다.
- 계절성·에너지 절감 캠페인 등 구조적 요인을 보정하기 위해 산업단지공단 가동률 동향 데이터를 보조 지표로 함께 투입한다.

**② Power-Risk Score (PRS) 산출**
- Z-score + 가동률 편차 + 업종별 계절 보정계수를 가중 합산하여 0~100점 PRS를 생성한다.
- PRS ≥ 70: 경보(RED) / PRS 50~69: 주의(YELLOW) / PRS < 50: 정상(GREEN) 3단계로 구분한다.
- 동일 지역 내 복수 업종이 동시에 RED 진입 시 '골목상권 복합위기' 알림을 추가로 발송한다.

**③ AI 이상탐지 방식 — 래퍼 이상의 독자 파이프라인**
- **이상탐지 모델**: Isolation Forest + LSTM Autoencoder를 전력사용 시계열에 적용한다. 업종별로 별도 학습하여 계절 패턴을 내재화한다.
- **매출 추정 레이어**: 전력사용량↔소상공인 매출액 간 업종별 상관계수를 소상공인시장진흥공단 상권정보시스템 공개 집계치[^1]와의 회귀분석으로 추정한다(`[추정]` — 실제 계수는 데이터 확보 후 보정 필요).
- **독자 자산**: 업종별 전력-매출 변환 계수 테이블, 지역별 계절 보정 룩업 테이블, 위기 탐지 임계값(PRS 산출 가중치)은 모델 운영 과정에서 누적·피드백 반영(데이터 네트워크 효과). 기반 ML 모델이 교체되어도 이 테이블과 피드백 루프 자산은 유지된다.
- **외부 LLM 활용(보조)**: 지원사업 텍스트를 임베딩하여 소상공인 업종·위기 유형에 맞는 지원사업을 RAG 방식으로 검색·추천한다. LLM은 추천 문구 생성에만 사용, 핵심 탐지 로직은 자체 ML 엔진에서 수행한다.

**④ 지원사업 자동매칭**
- 소상공인진흥공단·중소기업벤처부·지자체 공개 지원사업 DB를 크롤링하여 업종·매출규모·위기단계별 매칭 조건을 정형화한다.
- PRS 경보가 발생한 사업자의 업종·지역 조건과 매칭하여 즉시 신청 가능한 지원사업 목록을 반환한다.

## 2. 아이디어 구상 및 제안배경 (활용적정성)

### 2.1 해소하는 사회문제 — 소상공인 줄폐업·골목상권 붕괴

**폐업은 '사후 통계'로만 잡힌다 — 이것이 핵심 문제다.**

2023년 기준 소상공인 폐업 신고 건수는 약 **95만 건**(중소벤처기업부, 2023년 소상공인 실태조사)[^2]에 달한다. 하루 약 2,600여 개 점포가 문을 닫는 셈이다. 그러나 현행 지원 체계의 구조적 한계는 '폐업 등록 이후에야 지원 자격이 생긴다'는 점이다 — 폐업 전 예방 지원은 거의 없다.

국세청 사업자등록 통계, 행정안전부 지방행정데이터, 소진공 폐업 통계 모두 **이미 문을 닫은 뒤** 집계된다. 위기 징후가 수치로 잡힐 때는 이미 늦은 시점이다. 소상공인 본인도 자신의 업종·지역 상권 전체가 위기인지 알기 어렵고, 지원기관도 개별 사업체의 위기 수준을 사전에 파악할 수단이 없다.

소상공인 1개 업체 폐업 시 평균 부채는 약 **6,800만 원**(소상공인진흥공단, 2022)[^3]이다. 적시 지원으로 폐업을 1년 늦출 수 있다면, 부채 확대 억제·재기 지원 효율 개선·고용 유지 등 복합 효과가 발생한다. 소상공인 고용 인원은 약 **660만 명**(통계청 경제총조사, 2020)[^4]으로, 한 점포의 폐업은 평균 1~2명의 근로자 실직으로 이어진다.

**왜 지금인가 — 전력데이터가 선행지표가 되는 구조:**
- 소상공인 영업 활동은 전력소비와 직결된다(냉난방·조리·조명·간판·POS 등). 영업이 줄면 전력소비가 먼저 줄고, 줄폐업이 통계에 잡히는 것은 그 이후다.
- KEPCO가 2024년부터 산업분류별·계약종별 전력사용량을 data.go.kr에 공개 API로 제공하기 시작했다 — 이 공백이 막 열린 시점이다.
- 기존 상권 분석 서비스(소진공 상권정보시스템, 나이스비즈맵, KB부동산 상권분석 등)는 모두 '현황'을 보여주는 후행 지표(매출·유동인구·폐업수)에 의존한다. 전력소비를 '선행지표'로 쓴 상권 위기 조기경보 서비스는 존재하지 않는다.

### 2.2 활용분야·활용빈도·활용범위·중요성

**표 2.** 공공데이터 활용 4요소

| 요소 | 내용 |
|:---|:---|
| **활용분야** | 소상공인 경영지원·상권 분석·지역경제 정책 수립·폐업 예방 지원 |
| **활용빈도** | 월 1회 KEPCO API 갱신 주기에 맞춰 전국 시군구별 PRS 산출·갱신. 소상공인·지자체 대시보드는 상시 조회 가능(연간 약 12회 업데이트) |
| **활용범위** | 전국 시군구 228개 × 소상공인 밀집 업종 약 8개 = 약 1,824개 '업종×지역' 조합을 커버. 전국 소상공인 약 370만 개 사업체[^5]가 잠재 수혜 대상 |
| **중요성** | 폐업위기 조기 포착 → 선제 지원 연계 → 폐업 억제 → 골목상권 유지 → 지역 일자리 보호. 소상공인 부채·사회보험 미가입·재취업 어려움 등 2차 사회비용 절감 직결 |

### 2.3 경영혁신·창업학적 프레임워크

**Christensen 파괴적 혁신(Disruptive Innovation)** 관점에서, 버팀목은 기존 상권 분석 시장의 **비소비 영역(Non-consumption)**을 공략한다. 기존 서비스(나이스비즈맵·KB상권·소진공 상권정보)는 창업자·부동산 투자자·대형 가맹점본부가 주 고객이고, 이미 위기에 처한 소상공인 개인이 자신의 업종·지역 위험을 **사전에** 파악하는 서비스는 시장에 없다. 버팀목은 '공공데이터 + 저비용 ML 파이프라인'을 결합해 기존 유료 상권분석 서비스가 아예 커버하지 않던 시장을 저비용·고접근성으로 개척한다.

아울러 **JTBD(Jobs To Be Done)** 관점에서, 소상공인의 핵심 Job은 "사업이 힘들어지기 시작할 때 어떤 지원을 받을 수 있는지 알고 싶다"이며, 지자체·소진공의 Job은 "가장 위험한 상권과 사업체를 먼저 발굴해 자원을 효율 배분하고 싶다"이다. 버팀목은 이 두 Job을 하나의 서비스로 동시에 충족한다.

### 3.4 ④ 창의성·독창성

본 아이디어의 핵심 독창성은 **전력사용량을 상권 경기 선행지표로 재해석**한다는 점이다.

- 전력소비는 영업 활동과 거의 실시간으로 연동되며, 소상공인이 매출 감소를 인식하고 폐업을 결정하기 이전에 이미 소비 패턴 변화가 나타난다.
- 경제학에서 전력소비는 GDP·산업생산 선행지표로 널리 활용되지만[^6], 이를 '업종 × 지역 소상공인 단위'로 좁혀 **골목상권 폐업위기 조기경보**에 적용한 서비스는 국내외에서 확인되지 않는다.
- 데이터 공백이 아닌 **해석 공백**을 메우는 것이 창의성의 핵심이다 — 데이터(전력사용량)는 이미 공개되어 있고, 이를 소상공인 위기와 연결하는 인과 가설과 알고리즘이 신규 기여다.

**AI 활용 방식 구체 설명:**
1. **Isolation Forest**: 비지도 학습 앙상블로, '평년 전력소비 패턴'을 학습하고 그로부터 이탈하는 시점을 이상치로 탐지. 업종 × 지역 조합별로 독립 모델을 학습하여 계절·지역 특성을 내재화한다.
2. **LSTM Autoencoder**: 순환신경망 기반 시계열 압축·복원 모델. 정상 패턴을 학습한 뒤, 복원 오차가 임계값을 초과하는 달을 '이상 월'로 플래그한다. Isolation Forest와 앙상블하여 False Positive를 줄인다.
3. **RAG 기반 지원사업 추천**: 소상공인 업종·PRS 단계·지역을 쿼리로 변환하고, sentence-transformers로 임베딩한 지원사업 DB를 FAISS 벡터 인덱스에서 검색한다. LLM(경량 오픈소스 모델 또는 API)은 추천 결과 문구 생성에만 사용하며, 탐지 판단 자체는 수행하지 않는다.

| 소진공·지자체 상권모니터링 비용 | 인력 기반 현장조사·통계 집계 비용[추정] | 자동화로 모니터링 인력 투입 30% 절감[추정] | 대시보드 자동화 → 인력 현장 집중 재배치 |
| 커버 대상 소상공인 | 선제 위기 모니터링: 0명 | 전국 가동 시 약 370만 사업체 모니터링 대상[^5] | 전국 시군구 PRS 커버 |

> **[추정] 표기 안내**: 폐업률 감소 수치(5%p), 지원 연계 성공률(30%), 예산 효율화(20%p)는 유사 선행지표 활용 정책효과 사례를 참고한 추정값이며, 실증 파일럿 결과로 보정이 필요하다. 검증된 외부 수치는 각주[^2][^5]로 표기한다.

### 4.4 실현가능성 — 단계별 추진계획

**표 9.** 단계별 추진 로드맵

| 단계 | 기간 | 주요 활동 | 주요 산출물 |
|:---|:---|:---|:---|
| Phase 0. 기획·데이터 검증 | 0~3개월 | KEPCO API 실수집·전처리, 업종-전력 상관 분석, 파일럿 지자체 선정 | 데이터 분석 보고서, 파일럿 MOU |
| Phase 1. PoC·Beta | 3~9개월 | PRS 모델 1차 구축, 파일럿 5개 시군구 Beta 서비스 | Beta 대시보드, 파일럿 성과 보고 |
| Phase 2. 상용화·전국 확대 | 9~24개월 | 소상공인 셀프 조회 오픈, 30개 지자체 구독 전환 | 상용 서비스, SaaS 수익 발생 |
| Phase 3. 고도화 | 24~36개월 | 읍면동 단위 고해상도, 실시간 전력 AMI 데이터 연동[추정] | 버전 2.0, 정책 API 제공 |

**기술 리스크 대응**
- **리스크 1**: KEPCO 데이터가 시군구 단위 이하로 세분화 미제공 → 대응: 계약종별 데이터(15101360)와 교차 분석으로 소상공인 계약 구분 보완
- **리스크 2**: 전력-폐업 인과 상관이 약한 업종 존재 → 대응: 업종별 별도 보정계수 적용, 상관 낮은 업종은 경보 대상에서 제외하고 사용자에게 명시
- **리스크 3**: 공공기관 도입 의사결정 지연 → 대응: 무료 PoC → 성과 사례 확보 → 조달청 등록으로 구매 경로 단순화

### 4.5 차별화 기술의 구매동인 논증

**구매동인 가설**: 소진공·지자체의 핵심 의사결정 요인은 "한정된 지원 예산을 진짜 위기 사업체에 먼저 쓸 수 있는가"이다. 이 요인은 **must-have**에 가깝다 — 폐업 후 사업체를 지원하는 것(현재)보다 폐업 전 지원이 훨씬 효율적이라는 정책 방향이 이미 중기부·소진공 공식 목표로 설정되어 있기 때문이다[^7].

**가치 정량화 (추정)**: 소상공인 1개 사업체 폐업 방지 시 부채 6,800만 원 발생 억제[^3] + 고용 1~2인 유지 → 사회보험·실업급여 비용 절감. 지자체 입장에서는 지원예산 단위당 효과(비용효율) 향상이 직접 구매동인이다.

**반증·대안 위협**: 지자체가 인력·예산 부족으로 대시보드를 활용하지 않는 문제가 현실적 이탈 위험이다. 대응: 자동 보고서·알림 발송으로 담당자가 별도 조회 없이도 경보를 수신하는 '0 friction' 설계로 보완한다.
