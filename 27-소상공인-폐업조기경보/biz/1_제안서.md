# 버팀목 — 업종·지역 전력 둔화 기반 소상공인 폐업위기 조기경보·지원연계

> **아이디어 간략 개요 (3줄 이내)**
> 한국전력(KEPCO) 산업분류별·계약종별 전력사용량 공공데이터(15101403·15101360)를 업종×지역 단위로 집계·분석하여, 매출 둔화보다 먼저 나타나는 전력소비 이상 감소를 자동 포착하고, 위기 상권 소상공인에게 폐업 전 정부 지원사업을 선제 연계하는 조기경보·지원매칭 서비스다.
> 소상공인 영업 활동(냉난방·조리·조명·POS)은 전력소비와 직결되며, 폐업이 통계에 잡히기 3~6개월 전에 전력소비 이상 패턴이 먼저 나타난다[추정]; 이 시차를 역이용해 전력데이터를 '사전 위기 선행지표'로 활용한다.
> 위기 상권 지도·소상공인 대상 지원연계 알림·유관기관(소상공인시장진흥공단·지자체) 전용 대시보드를 3개 레이어로 제공하며, 전국 약 370만 소상공인 사업체를 잠재 수혜 대상으로 한다[^5].

**핵심 기술·서비스·정보 명칭**
- 전력기반 매출둔화 추정 모델 (전력소비 이상탐지 + 업종별 보정 알고리즘)
- 업종·지역 폐업위험 조기경보 스코어 (Power-Risk Score, PRS)
- 소상공인 지원사업 자동매칭 엔진 (RAG 기반 벡터 검색)
- 위기 상권 대시보드 (소진공·지자체 전용 / 소상공인 셀프 조회)

## 1. 아이디어 기획 핵심내용 (구체성, 우수성)

### 1.1 무엇을 만드는가 — 서비스 개요

**버팀목**은 한국전력(KEPCO)이 공개하는 산업분류별·계약종별 전력사용량 공공데이터를 핵심 원천으로, 업종(음식점·소매·숙박 등)과 행정구역(시군구 단위 확정, 읍면동 단위는 Phase 3[추정]) 교차로 월별 전력소비 추세를 추적한다. 전력소비가 해당 업종·지역 평년 대비 설정 임계값 이하로 떨어지면 '위기 상권'으로 플래그하고, 소상공인진흥공단·지자체·소상공인 본인에게 각각 맞춤 알림과 지원연계 정보를 제공한다.

### 1.2 핵심 구현 내용

**① 전력기반 위기 신호 포착**

KEPCO 산업분류별 전력사용량(데이터셋 15101403)과 계약종별 전력사용량(15101360)을 월별 API로 수집한다. 두 데이터셋의 교차 활용이 핵심이다: 산업분류별 데이터는 한국표준산업분류(A~Q) 대분류로 소매(G)·음식숙박(I)·기타서비스(S) 등 소상공인 밀집 업종을 식별하고, 계약종별 데이터는 일반용 갑/을·소규모 계약 구분으로 소상공인 계약 유형을 특정한다. 시군구 행정코드와 결합하여 '업종군 × 시군구' 매트릭스(약 1,824개 셀 = 228개 시군구 × 8개 업종군)를 구성하고, 최근 12개월 이동평균 대비 직전 2개월 편차를 Z-score로 환산한다. 계절성·에너지절감 캠페인 등 구조적 요인을 보정하기 위해 한국산업단지공단(KICOX) 국가산단 동향(15085894·15085886 계열) 가동률 데이터를 보조 지표로 투입한다[추정 — KICOX 데이터셋 ID는 산업단지 유형에 따라 상이할 수 있어 실수집 시 확인 필요].

**② Power-Risk Score (PRS) 산출 공식**

PRS = Z-score × 0.6 + 가동률편차 × 0.2 + 계절보정 × 0.2 (가중치는 [추정]; 실증 파일럿 후 최소제곱 회귀로 보정 예정)

- PRS ≥ 70: 경보(RED) — 즉시 지원사업 매칭·알림 발송
- PRS 50~69: 주의(YELLOW) — 대시보드 강조 표시·월 1회 알림
- PRS < 50: 정상(GREEN) — 모니터링 계속
- 동일 지역 내 복수 업종이 동시에 RED 진입 시 '골목상권 복합위기' 추가 알림 발송

**③ AI 이상탐지 파이프라인 — 래퍼 이상의 독자 엔진**

- **Isolation Forest(비지도)**: 업종군 × 시군구 조합별로 독립 모델을 학습하여 계절 패턴·지역 특성을 내재화한다. '평년 전력소비 패턴'으로부터 이탈 정도를 이상 점수(anomaly score)로 산출한다.
- **LSTM Autoencoder(시계열)**: 순환신경망 기반으로 정상 패턴을 학습한 뒤, 복원 오차(reconstruction error)가 업종별 동적 임계값을 초과하는 달을 '이상 월'로 플래그한다. Isolation Forest와 앙상블(다수결)로 False Positive를 절감한다.
- **독자 자산(모델 교체 불가)**: 업종별 전력-매출 변환계수 테이블, 지역별 계절 보정 룩업 테이블, PRS 가중치는 서비스 운영 과정에서 누적·피드백 반영(데이터 네트워크 효과). 기반 ML 라이브러리가 교체되어도 이 테이블과 피드백 루프 자산은 보존된다.
- **외부 LLM 활용(보조)**: 지원사업 텍스트를 sentence-transformers로 임베딩하여 소상공인 업종·위기 유형에 맞는 지원사업을 RAG(Retrieval-Augmented Generation) 방식으로 검색·추천한다. LLM은 추천 문구 생성에만 사용; 핵심 탐지·스코어링 로직은 자체 ML 엔진이 수행한다.
- **전력-매출 상관 레이어**: 소상공인시장진흥공단 상권정보시스템 공개 집계치와의 회귀분석으로 업종별 전력-매출 변환계수를 추정한다 (`[추정]` — 실계수는 데이터 확보 후 보정).

**④ 지원사업 자동매칭**

소상공인진흥공단·중소벤처기업부·지자체 공개 지원사업 DB(https://www.bizinfo.go.kr)를 크롤링하여 업종·매출규모·위기단계별 매칭 조건을 정형화한다. PRS 경보 발생 시 해당 업종·지역 조건과 즉시 매칭하여 신청 가능 지원사업 목록을 반환한다.

## 2. 아이디어 구상 및 제안배경 (활용적정성)

### 2.1 해소하는 사회문제 — 소상공인 줄폐업·골목상권 붕괴

**폐업은 '사후 통계'로만 잡힌다 — 이것이 핵심 구조적 공백이다.**

2023년 기준 소상공인 폐업 신고 건수는 약 **95만 건**(중소벤처기업부, 2023년 소상공인 실태조사)[^2]이다. 하루 약 2,600여 개 점포가 문을 닫는 셈이다. 전국 소상공인 사업체 약 370만 개[^5] 대비 연간 폐업률은 약 25%에 달한다. 그러나 현행 지원체계의 구조적 한계는 '폐업 등록 이후에야 지원 자격이 생긴다'는 점이다 — 폐업 전 예방 지원은 거의 없다.

국세청 사업자등록 통계, 행정안전부 지방행정데이터, 소진공 폐업 통계 모두 **이미 문을 닫은 뒤** 집계된다. 위기 징후가 수치로 잡힐 때는 이미 늦은 시점이다. 소상공인 본인도 자신의 업종·지역 상권 전체가 위기인지 알기 어렵고, 지원기관도 개별 사업체의 위기 수준을 사전에 파악할 수단이 없다.

소상공인 1개 업체 폐업 시 평균 부채는 약 **6,800만 원**(소상공인진흥공단, 2022)[^3]이다. 소상공인 고용 인원은 약 **660만 명**(통계청 경제총조사, 2020)[^4]으로, 한 점포의 폐업은 평균 1~2명 근로자의 실직으로 이어진다. 2023년 폐업 95만 건 기준으로 연간 최대 **190만 명**의 고용 기회가 소실될 수 있다[추정].

소상공인 지원 예산은 중기부·소진공 합산 연간 약 **3조 원 이상**[추정; 중기부 예산 공개 기준 재확인 필요]이나, 이 예산이 이미 폐업한 사업체의 재기 지원에 집중되는 구조다. 폐업 전 선제 개입이 가능하다면 동일 예산으로 훨씬 높은 효율을 낼 수 있다.

**왜 지금인가 — 전력데이터가 선행지표가 되는 구조:**
- 소상공인 영업 활동은 전력소비와 직결된다(냉난방·조리·조명·간판·POS 단말기 등). 영업이 줄면 전력소비가 먼저 줄고, 줄폐업이 통계에 잡히는 것은 3~6개월 뒤다[추정].
- KEPCO가 2024년부터 산업분류별·계약종별 전력사용량을 data.go.kr에 공개 API로 제공하기 시작했다 — 이 공백이 막 열린 시점이다.
- 기존 상권 분석 서비스(소진공 상권정보시스템·나이스비즈맵·KB상권 등)는 모두 '현황'을 보여주는 후행 지표(매출·유동인구·폐업수)에 의존한다. 전력소비를 '선행지표'로 쓴 상권 위기 조기경보 서비스는 국내에 존재하지 않는다.

| **활용범위** | 전국 시군구 228개 × 소상공인 밀집 업종군 약 8개 = 약 1,824개 '업종×지역' 조합 커버. 전국 소상공인 약 370만 사업체[^5]가 잠재 수혜 대상 |

### 2.3 경영혁신·창업학적 프레임워크

**Christensen 파괴적 혁신(Disruptive Innovation)** 관점에서, 버팀목은 기존 상권 분석 시장의 **비소비 영역(Non-consumption)**을 공략한다. 기존 서비스(나이스비즈맵·KB상권·소진공 상권정보)는 창업자·부동산 투자자·대형 가맹점본부가 주 고객이고, 이미 위기에 처한 소상공인 개인이 자신의 업종·지역 위험을 **사전에** 파악하는 서비스는 시장에 없다. 버팀목은 '공공데이터 + 저비용 ML 파이프라인'을 결합해 기존 유료 상권분석 서비스가 아예 커버하지 않던 시장을 저비용·고접근성으로 개척한다.

아울러 **JTBD(Jobs To Be Done)** 관점에서, 소상공인의 핵심 Job은 "사업이 힘들어지기 시작할 때 어떤 지원을 받을 수 있는지 알고 싶다"이며, 지자체·소진공의 Job은 "가장 위험한 상권과 사업체를 먼저 발굴해 자원을 효율 배분하고 싶다"이다. 버팀목은 이 두 Job을 하나의 서비스로 동시에 충족한다.

### 3.4 ④ 창의성·독창성

본 아이디어의 핵심 독창성은 **전력사용량을 상권 경기 선행지표로 재해석**한다는 점이다.

- 전력소비는 영업 활동과 거의 실시간으로 연동되며, 소상공인이 매출 감소를 인식하고 폐업을 결정하기 이전에 이미 소비 패턴 변화가 나타난다.
- 경제학에서 전력소비는 GDP·산업생산 선행지표로 널리 활용되지만[^6], 이를 '업종 × 지역 소상공인 단위'로 좁혀 **골목상권 폐업위기 조기경보**에 적용한 서비스는 국내외에서 확인되지 않는다.
- 데이터 공백이 아닌 **해석 공백**을 메우는 것이 창의성의 핵심이다 — 데이터(전력사용량)는 이미 공개되어 있고, 이를 소상공인 위기와 연결하는 인과 가설과 알고리즘이 신규 기여다.

**AI 활용 방식 구체 설명:**

1. **Isolation Forest**: 비지도 학습 앙상블로 '평년 전력소비 패턴'을 학습하고, 그로부터 이탈하는 시점을 이상치로 탐지한다. 업종군 × 시군구 조합별로 독립 모델을 학습하여 계절·지역 특성을 내재화한다. 학습 데이터는 최근 24개월 이상 전력사용량 시계열이며, 모델 갱신은 월 1회 배치로 수행한다.

2. **LSTM Autoencoder**: 순환신경망 기반 시계열 압축·복원 모델. 정상 패턴을 학습한 뒤 복원 오차(reconstruction error)가 업종별 동적 임계값(μ + 2σ [추정])을 초과하는 달을 '이상 월'로 플래그한다. Isolation Forest와 앙상블(소프트 보팅)하여 False Positive를 절감한다.

3. **RAG 기반 지원사업 추천**: 소상공인 업종·PRS 단계·지역을 쿼리로 변환하고, sentence-transformers로 임베딩한 지원사업 DB를 FAISS 벡터 인덱스에서 검색한다. LLM(경량 오픈소스 모델 또는 외부 API)은 추천 결과 문구 생성에만 사용하며, 탐지 판단 자체는 수행하지 않는다. 지원사업 DB는 월 1회 갱신하며, 새 공고가 즉시 벡터 인덱스에 추가된다.

**서비스 방법:**
- **Phase 1 (0~9개월)**: 5개 파일럿 시군구 선정 → 소진공 지역센터와 MOU → 대시보드 Beta 제공 → 파일럿 성과 사례화
- **Phase 2 (9~24개월)**: 전국 확대 → 소상공인 셀프 조회 오픈 → 카카오 알림톡[추정] 연동 → 30개 지자체 SaaS 전환
- **Phase 3 (24~36개월)**: 지자체 정책 연동 API 제공 → 읍면동 단위 고해상도 탐지 → AMI 실시간 데이터 연동[추정]

**그림 4.** 버팀목 수익모델 — 소상공인 무료·공공기관 SaaS·데이터 용역 3원 구조

**수익원 및 가격 정책**

| 수익원 | 가격 | 설명 |
|:---|:---|:---|
| 지자체·소진공 SaaS 구독 | 시군구당 월 30~70만 원[추정] | 대시보드·API·월별 보고서 포함 |
| 데이터 분석 용역 | 프로젝트당 1,000~5,000만 원[추정] | 특정 상권 심층 분석·정책 수립 지원 |
| 소상공인 셀프 조회 | **무료** | 공공성 확보·사용자 기반 확대 |

**단위경제성 (추정)**

| 지표 | 수치 | 산출 근거 |
|:---|:---|:---|
| CAC (지자체 1곳 영업·PoC 비용) | 약 300만 원[추정] | PM 1인 2개월 + 데모 인프라 비용 |
| LTV (지자체 구독 연 360~840만 원 × 평균 3년) | 1,080~2,520만 원[추정] | 구독 단가 × 유지 기간 |
| LTV / CAC | **3.6~8.4배**[추정] | LTV ÷ CAC |
| 손익분기 (BEP) | 구독 지자체 약 15~20곳[추정] | 서버·인건비 월 500~700만 원 대비 |
| 초기 CAC 절감 방법 | 소진공 공동사업 MOU → 무료 PoC 선제 제공 | 영업비용을 소진공 채널로 대체 |

> **LTV/CAC 3.6~8.4배**: 일반 SaaS 기준 LTV/CAC 3배 이상이 지속가능 모델이므로[추정], 하한 3.6배도 기준 충족. 단, 초기 파일럿 단계에서는 무료 PoC 제공으로 CAC가 일시적으로 높아질 수 있다 — 소진공 채널 공동사업으로 절감 필요.

**고객확보(GTM) 전략**

- **초기 (첫 5개 지자체)**: 소상공인시장진흥공단 지역센터와 공동사업 협약 → 무료 PoC 제공 → 성과 사례화 → 타 지자체 레퍼런스 영업
- **확산 (6~30개)**: 지자체 정보화담당부서 직접 영업 + 소진공 전국 조달 활용[추정]
- **소상공인 채널**: 소진공 앱·문자·카카오채널 연동으로 알림 발송 → 셀프 조회 진입

**차별성·경쟁우위(Moat)**
- **데이터 선점**: KEPCO 전력데이터 + 소상공인 피드백 데이터가 쌓일수록 모델 정확도 향상 (데이터 네트워크 효과)
- **공공 파트너십**: 소진공·지자체 협약이 민간 경쟁사의 진입 장벽
- **전환비용**: 지자체가 대시보드를 정책 워크플로에 통합하면 교체 마찰이 크다

**Why now**: KEPCO 전력데이터 API가 2024년부터 공개되었고, 소상공인 폐업 문제가 2023~2025년 사회 어젠다로 부각되는 시점이 맞물린다. 기존 상권분석 서비스의 '선행지표 공백'이 채워지지 않은 지금이 진입 최적기다.

### 4.5 차별화 기술의 구매동인 논증

**구매동인 가설**: 소진공·지자체의 핵심 의사결정 요인은 "한정된 지원 예산을 진짜 위기 사업체에 먼저 쓸 수 있는가"이다. 이 요인은 **must-have**에 가깝다 — 폐업 후 지원보다 폐업 전 지원이 훨씬 효율적이라는 정책 방향이 중기부·소진공 공식 목표로 이미 설정되어 있기 때문이다[^7].

**가치 정량화 (추정)**: 소상공인 1개 사업체 폐업 방지 시 부채 6,800만 원 발생 억제[^3] + 고용 1~2인 유지 → 사회보험·실업급여 비용 절감. 지자체 입장에서는 지원예산 단위당 효과(비용효율) 향상이 직접 구매동인이다. 시군구당 SaaS 비용 연 360~840만 원 대비 폐업 방지 1건의 사회적 가치 6,800만 원[^3]이면 ROI는 8~19배[추정]에 달한다.

**외부 근거**: 중기부는 2024년 「소상공인 폐업 예방 지원 강화 방안」에서 폐업 전 선제 지원을 공식 정책 목표로 명시했다[^7]. 소진공 예산의 '경영개선' 항목이 매년 증가하는 추세는 이 구매동인의 정책 수요를 실증한다[추정; 소진공 예산서 공개 기준 재확인 필요].

**반증·대안 위협**: 지자체가 인력·예산 부족으로 대시보드를 활용하지 않는 문제가 현실적 이탈 위험이다. 대응: 자동 보고서·알림 발송으로 담당자가 별도 조회 없이도 경보를 수신하는 '0 friction' 설계로 보완한다. 또한 공공기관의 데이터 수집·활용 거버넌스 이슈(개인정보·영업비밀 우려 등)가 도입 장벽이 될 수 있다 — 업종×지역 집계치(개별 사업체 데이터 아님)만 활용함을 명시하여 개인정보 우려를 원천 차단한다.
