# k-startup-template — 공모전·창업지원사업 산출물 템플릿 키트

공모전·창업지원사업·연구과제의 **아이디어 제안서 + 조사자료**를 규약대로(정량 기준·논문형 그림·정직성) 만들기 위한 **재사용 템플릿**이다. **Claude Code 전용**으로 운영한다.

> ⛔ **기본 모드 (CLAUDE.md §5)**: 평시 산출물은 **아이디어별 프로젝트 디렉터리 + 제안서 + 조사(`5_research/`)까지**다. 개발(개발계획서·과업지시서·개발결과보고서·`projects/` 데모 앱·v1/v2)은 **사용자의 별도 개발 지시가 있을 때만** 진행한다. 본 키트는 순수 아이디어 평가를 기본 전제로 한다.

## 이렇게 쓴다 (배포 모델)

```
①  이 레포를 clone (또는 워크스페이스로 사용)
②  (선택) MCP 서버 등록 — 웹검색으로 조사 강화
③  공고 PDF 한 개를 레포에 넣는다 (또는 한 줄 아이템)
④  Claude Code 에:  "CLAUDE.md 읽고 진행해"
⑤  아이디어별 프로젝트 디렉터리 + 제안서 + 조사를 알아서 채운다
```

무엇을·어떤 기준으로·어떤 순서로 만들지는 전부 레포 안의 규칙 파일([`CLAUDE.md`](CLAUDE.md)·[`.claude/agents/`](.claude/agents/))에 들어 있다.

## 구성

| 경로 | 내용 |
|:---|:---|
| [`CLAUDE.md`](CLAUDE.md) | **규칙 단일 출처** — 디렉터리 규약·문서 규약·평가기준 정합·정량 기준·DoD·금지사항 |
| [`.claude/agents/`](.claude/agents/) | 역할 정의: proposal·research·차별점·figure·qa·orchestrator (+ 개발 지시 시 app·capture·report) |
| [`design-guide/`](design-guide/) | (개발 지시 시) 데모 앱 디자인 가이드(애플×토스·모바일·PC·반응형 검수) |
| [`그림자료_규약.md`](그림자료_규약.md) | 문서 그림 = 순수 흑백 논문형 Mermaid init·변환 절차(단일 출처) |
| [`forms/`](forms/) | 문서 양식 + 참고자료 — `forms/biz/`(빈 골격)·`forms/refs/`(참고 PDF) |

## 핵심 정량 기준 ([CLAUDE.md](CLAUDE.md))
- 참고문헌 **≥ 1,000** (실제 출처만, 날조·중복 금지)
- 차별점 **≥ 50** (카테고리화, 부풀리기 금지)
- 문서 그림자료 = **논문형 순수 흑백**(컬러·회색 0, 번호·캡션)
- *(개발 지시 후)* 데모 앱 기능 **≥ 100**(실 동작만)·**모바일·PC 반응형**·버전 브랜치

# 사용법 (Usage)

## 1. 출발 씨앗 넣기 (기획의 시작점)

작업은 **공고 PDF**(가장 정확) 또는 **한 줄 아이템**에서 출발한다. 둘 다 없으면 키트 예시 공고로 데모한다.

- **공고 PDF**: 공고(과제) PDF 를 레포 루트에 `공고-<사업명>.pdf` 로 두거나 [`forms/refs/`](forms/refs/) 에 넣는다.
- **한 줄 아이템**: 파일 없이 명령에 아이템을 적어도 된다(예: `"대학생 중고거래 SaaS"`).

기획 부트스트랩이 공고에서 메타(사업명·주관기관·트랙·일정·섹션 순서)를 **공고에 적힌 사실만** 뽑아 `<프로젝트>/biz/` 를 만들고 제안서 머리표까지 채운 뒤 진행한다(없는 한도·자격은 창작하지 않음).

## 2. 실행 — "읽고 진행해"

Claude Code 에서:

```
CLAUDE.md 읽고 진행해
```

또는 병렬 자율 루프로 끝까지:

```
orchestrator 로 자율 루프 돌려서 제안서 + 조사까지 끝까지 완성해
```

오케스트레이터가 역할 에이전트(`proposal-writer`·`research-collector`·`differentiation-analyst`·`figure-maker`·`qa-auditor`)를 병렬 spawn 하고 검증·커밋까지 일괄 수행한다. **개발 역할(app-builder·capture-runner·report-writer)은 사용자가 "개발 진행"을 지시하기 전까지 spawn 하지 않는다.**

## 3. (선택) MCP 서버 등록 — 조사 강화

웹검색 MCP 를 붙이면 `research-collector` 의 참고문헌 수집이 안정적이다. **API 키는 `.env`/셸 env 로, 레포에 커밋 금지.**

```bash
claude mcp add tavily --env TAVILY_API_KEY=$TAVILY_API_KEY -- npx -y tavily-mcp
claude mcp list
```

## 4. 무엇이 일어나나 (자율 루프)

ASSESS → PLAN → EXECUTE → VERIFY → REPEAT 를 게이트 모두 충족까지 반복한다([`CLAUDE.md` §9](CLAUDE.md)).
- **전면 재검토** — 이미 만든 산출물도 매 사이클 *처음 관점*에서 다시 의심·실측해 끌어올린다(증분 점검 아님).
- **정직성 우선** — 정량 미달을 허위 충족으로 적지 않고 `현재/목표` 수치를 남긴 채 더 돈다.
- **멈추지 않음** — 한 항목이 끝나면 다음 미달 항목으로 자동 진행(매번 확인 안 함).

산출물은 `<프로젝트>/biz/`(제안서)·`<프로젝트>/biz/5_research/`(조사)에 쌓인다. *(개발 지시 후에는 `projects/`·`biz/captures/` 가 추가된다.)*

## 5. 사람이 채울 것

팀·서명·연락처 등 행정 빈칸은 자동으로 안 채운다 — `<TODO: 사용자 입력>` 로 남는다. 제출 전 [`forms/biz/_사용자입력_필요항목.md`](forms/biz/_사용자입력_필요항목.md) 기준으로 직접 채운다.

# 트러블슈팅

| 증상 | 원인 / 해결 |
|:---|:---|
| 규칙을 안 따르는 듯 | `CLAUDE.md` 가 규칙 단일 출처다. "CLAUDE.md 읽고 진행해" 로 시작한다. |
| **토큰을 적게 쓰고 금방 끝남 = 얕게 함** | 작업은 방대해서(참고문헌 1,000+·차별점 50+) 짧게 끝날 수 없다. CLAUDE.md §9 가 골격·"나중에 계속"·미실측 ✅ 를 금지하고 게이트를 grep/카운트로 실측하게 한다. |
| 참고문헌 1,000 이 한 번에 안 참 | 정상 — 컨텍스트를 넘기므로 `현재/목표` 수치를 남기고 멈춘다. 다시 실행하면 남은 분량부터 이어간다. |
| 공고 PDF 를 못 읽음 | `brew install poppler`(`pdftotext`) 설치, 또는 검색/파일 MCP 사용. |
| 문서 그림이 회색/컬러로 나옴 | [`그림자료_규약.md`](그림자료_규약.md) 의 Mermaid base init 사용 + `mmdc` 렌더 눈검수. |

규칙 전체는 [`CLAUDE.md`](CLAUDE.md).
