# 모바일 디자인 가이드 (Mobile, ≤767px)

> 본 문서는 이 워크스페이스의 **모든 웹 데모(`*/projects/*`)가 모바일(좁은 화면 ≈360~430px)에서 지켜야 할 레이아웃·인터랙션 규칙**이다.
> 상위 톤 규칙은 [디자인_애플_토스.md](./디자인_애플_토스.md)를, 데스크톱 규칙은 [디자인_PC.md](./디자인_PC.md)를, **출고 전 필수 검수**는 [반응형_검수_필수.md](./반응형_검수_필수.md)를 따른다.
>
> ⚠️ **가장 중요한 원칙:** 모바일은 PC 레이아웃의 축소판이 아니다. **레이아웃을 다시 짠다.** `overflow=0`·`pageerror=0`은 "잘 보인다"의 증거가 **아니다** — 반드시 390px 스크린샷을 **눈으로** 본다([반응형_검수_필수.md](./반응형_검수_필수.md)).

---

## 0. 절대 금지 (지금까지 실제로 터진 실패들)

| ❌ 금지 | 왜 | ✅ 대신 |
|:---|:---|:---|
| 데스크톱 사이드바(`w-60`/`w-64`)를 모바일에서 그대로 노출 | 화면 절반(240~256px)을 먹고 본문이 130px로 짓눌림 | 사이드바 숨기고 **바텀 탭바** 또는 **햄버거 오프캔버스 드로어** |
| `viewport width=1280/1440` 데스크톱 고정으로 "모바일 대응" 갈음 | 모바일에서 잘리거나 잘게 축소돼 판독 불가 | `width=device-width, initial-scale=1` + 진짜 반응형 |
| 사이드바를 모바일에서 `max-height:46vh` 상단 패널로 전환 | 어두운 패널이 화면 위 절반을 하이재킹, 콘텐츠가 한참 아래 | 오프캔버스 드로어(화면 밖 대기, 햄버거로 호출) |
| `grid-cols-3`/`grid-cols-4` 고정 | 카드가 손가락 너비로 쪼개지고 숫자 잘림 | `grid-cols-2`(또는 1열) 모바일 스택 |
| 차트 `<canvas>`를 폭 0으로 수렴시킴 | Chart.js가 0×0으로 붕괴, 빈 차트 | 고정 높이 래퍼 + `maintainAspectRatio:false` |
| 탭 타깃 <44px | 손가락으로 못 누름 | 주요 액션 ≥44px(이상적 52px) |

---

## 1. 뷰포트·기본 골격

- `<meta name="viewport" content="width=device-width, initial-scale=1">` **필수**. 고정폭 viewport 금지.
- 좁은 화면 **가로 overflow 0**. 단 `overflow:hidden`으로 잘라서 0을 만드는 건 위장 — 콘텐츠가 풀폭으로 **다 보여야** 한다.
- 기준 폭: **390×844**(iPhone 12/13/14). 360px(소형 안드로이드)에서도 안 깨지게.
- 본문 콘텐츠는 **풀폭**(좌측 사이드바 마진 0). 문서형 본문은 중앙 정렬 + 좌우 패딩 16px.

---

## 2. 내비게이션 — 둘 중 하나 (사이드바 그대로 금지)

### 2.1 바텀 탭바 (권장: 목적지 3~5개로 좁힐 수 있을 때)
- 화면 하단 고정(`fixed bottom-0 inset-x-0`), 5개 내외 아이콘+라벨, 각 버튼 **min-height 44px**.
- 목적지가 5개를 넘으면 마지막 칸을 **"더보기(⋯)"** 로 두고 바텀시트에 나머지 전체 메뉴(RBAC 있으면 권한 내) 노출.
- 본문 하단에 바텀바 높이만큼 `padding-bottom`(예 5rem)으로 가림 방지.
- 데스크톱(≥768px)에선 바텀바 숨기고 사이드바 노출: `@media(max-width:767px){#sidebar{display:none}#bottomnav{display:flex}} @media(min-width:768px){#bottomnav{display:none}}`.

### 2.2 햄버거 오프캔버스 드로어 (메뉴가 많은 대시보드)
- 사이드바를 기본 화면 밖에: `position:fixed; inset-block:0; left:0; width:82vw; max-width:300px; transform:translateX(-100%); transition:transform .25s;`.
- **모바일 상단 앱바**(`@media(min-width:768px){display:none}`): 좌측 햄버거 ☰ + 타이틀/브랜드.
- 햄버거 → 드로어 슬라이드 인 + **반투명 백드롭**(잉크 0.4). 백드롭/메뉴 항목 클릭 → 닫힘.
- 드로어 열림 상태는 클래스 토글(`body.drawer-open`) 또는 React `useState`로.

> 어떤 방식이든 **데스크톱(≥768px) 레이아웃은 무변경**(사이드바 그대로). 모바일 분기만 추가한다.

---

## 3. 콘텐츠 레이아웃

- **그리드 스택:** `grid-cols-1 sm:grid-cols-2 md:grid-cols-4` 식. KPI 4개 → 모바일 2열. 카드 비율이 너무 납작/길쭉하지 않게.
- **표(table):** 모바일에서 통째로 줄이지 말고 **가로 스크롤 컨테이너**(`overflow-x-auto`)로 감싸거나, 카드형 리스트로 재구성. 표가 잘려 일부 열이 사라지면 안 됨.
- **차트:** `<canvas>`를 **고정 높이 래퍼**(예 `h-56`)로 감싸고 Chart.js `responsive:true, maintainAspectRatio:false`. 모바일에서 실제로 그려지는지 스크린샷 확인.
- **모달/선택/확인:** 모바일은 중앙 모달보다 **바텀시트** 우선(엄지 도달). frosted glass는 떠 있는 레이어에만([디자인_애플_토스.md](./디자인_애플_토스.md) §6).
- **타이포:** 본문 최소 15~16px(읽기), 캡션 12px+. 제목이 한 줄에 안 들어가면 `whitespace-nowrap`+요소 축약 또는 줄 구조 재배치(글자가 세로로 한 자씩 쪼개지면 실패).

---

## 4. 터치·인터랙션

- **탭 타깃 ≥44px**(주요 액션·풀폭 버튼은 52px). 헤더 보조 버튼·칩·셀렉트도 44px.
- 눌림 피드백 `scale(0.97)`, 비활성은 브랜드 `opacity:.35`.
- 좌우 끝 버튼은 가장자리에서 8px+ 띄워 오탭 방지.
- 가로 스와이프 영역과 페이지 스크롤 충돌 주의.

---

## 5. 캡처(검수 증거)

- 모바일 캡처는 **390×844(isMobile, hasTouch)** 로 별도 폴더: `captures/mobile/<vN>/`.
- 최소 3컷: `01_home`(상단), `02_scroll`(스크롤된 본문), `03_action`(핵심 인터랙션·드로어 열림 등).
- **캡처를 만든 사람이 직접 그 PNG를 열어 비율을 확인**한다([반응형_검수_필수.md](./반응형_검수_필수.md)).
