# forms/ — 문서 양식 + 참고자료 (단일 보관 디렉터리)

문서 양식·참고자료는 **전부 이 디렉터리 하나**에 모은다. 새 양식·참고문서가 생기면 여기에 넣는다.

| 경로 | 내용 |
|:---|:---|
| [`biz/`](biz/) | 양식파일(빈 골격) — 제안서·개발계획서·과업지시서·결과보고서·research·사용자입력시트 |
| [`정산양식/`](정산양식/) | 지원금 정산 구비서류(빈 골격) — 거래명세서·검수조서·외주용역 결과보고서·지출기준 |
| [`refs/`](refs/) | 참고 PDF + 변환 마크다운(공고문·사업계획서 예시·거래명세서 양식 등) — 목차·표 구조·톤만 참고 |

규칙·자율 작성 루프는 모두 레포 루트 [`CLAUDE.md`](../CLAUDE.md)(단일 출처).

새 프로젝트 시작 시 `biz/` 골격을 `<project>/biz/` 로 복사해 출발점으로 쓴다.

|:---|:---|
| `biz/1_제안서.md` | CLAUDE.md §2.1 (프레임워크·GTM·수익·차별성 50+·구매동인·참고문헌 1,000+) |
| `biz/2_개발계획서.md` | §2.2 (WBS 자체/외주) |
| `biz/3_과업지시서_v1.md` | §2.3 (5섹션) |
| `biz/3_1_개발결과보고서_v1.md` | §2.4 (D구조, 기능 100+ 목록표) |
| `biz/5_research/README.md` | §2.5 (출처 1,000+) |
| `biz/_사용자입력_필요항목.md` | §2.7 (팀·서명 빈칸) |

v2/v3 는 `4_과업지시서_v2.md`·`4_1_개발결과보고서_v2.md`·`5_과업지시서_v3.md`·`5_1_개발결과보고서_v3.md`

## refs/ 참고 PDF
실제 작업 워크스페이스에서는 [`CLAUDE.md` §1](../CLAUDE.md) 대로 참고 PDF 를 워크스페이스 루트(`ref-*.pdf`)에 둔다.
여기 `forms/refs/` 는 **키트가 보관하는 원본** — 새 작업 시작 시 워크스페이스 루트로 복사해 쓴다.
각 PDF 는 동명의 `.md`(opendataloader-pdf 변환본)를 함께 보관해 본문 인용에 쓴다(빈 양식 PDF 는 표 미인식 → 직접 복원).
